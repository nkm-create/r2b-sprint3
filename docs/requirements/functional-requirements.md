### F011: 充足率サマリー表示

| 項目名 | 計算式 |

### F012: 人員状況ヒートマップ表示

曜日×限のマトリクスで需給状況を色分け表示。

#### 需給差の定義

各時間枠（曜日×限）における「講師が提供可能な授業枠数」と「生徒の受講希望人数」のバランスを示す指標。

| 項目 | 計算式 |
|------|--------|
| 需給差 | 供給枠数 - 需要人数 |
| 供給枠数 | その枠で勤務可能な講師数 × 2（1対2形式想定） |
| 需要人数 | その枠で受講可能（○または△）の生徒数 |

**解釈**: 「その枠であと何人の生徒を受け入れ可能か」を示す。

| 需給差 | ステータス | 表示色 |
|--------|-----------|--------|
| >= 2 | 余裕あり | 緑 |
| 1 | 適正 | 黄緑 |
| 0 | ギリギリ | 黄 |
| < 0 | 不足 | 赤 |

**データソース**:
- 供給: 講師シフト希望（◯の講師数）
- 需要: 生徒受講希望（○/△の生徒数）

### F013: 科目別カバー率表示

科目ごとに「その科目の授業を十分に提供できる講師がいるか」を示す指標。

#### この指標で知りたいこと

「数学を週40コマ分教えてほしい生徒がいるけど、数学を教えられる講師は足りている？」という問いに答える。

#### 計算の考え方

##### ステップ1: 需要を集計する

生徒マスタの「科目別コマ数」から、科目ごとの需要コマ数を合計する。

**例**:
| 生徒 | 科目別コマ数 |
|------|-------------|
| 生徒A | 数学:2 |
| 生徒B | 数学:1, 英語:2 |
| 生徒C | 英語:3 |

→ 数学の需要 = 2 + 1 = **3コマ**
→ 英語の需要 = 2 + 3 = **5コマ**

##### ステップ2: 供給を集計する（按分方式）

講師マスタの「最大コマ/週」から供給を計算する。ただし、1人の講師が複数科目を教えられる場合、そのまま足すと二重カウントになる問題がある。

**悪い例（二重カウント）**:
| 講師 | 教えられる科目 | 最大コマ/週 |
|------|--------------|------------|
| 講師X | 数学, 英語 | 4 |

→ 数学の供給 = 4コマ、英語の供給 = 4コマ？
→ 合計8コマ供給できることになるが、講師Xは週4コマしか働けない。**おかしい。**

**按分方式（正しい計算）**:

講師Xの4コマを、数学と英語の需要の比率で分ける。

- 数学の需要 = 3コマ
- 英語の需要 = 5コマ
- 合計 = 8コマ

講師Xの供給の配分:
- 数学への供給 = 4 × (3 ÷ 8) = **1.5コマ**
- 英語への供給 = 4 × (5 ÷ 8) = **2.5コマ**

これなら合計4コマで、講師Xの実際の働ける量と一致する。

##### ステップ3: カバー率を計算する

```
科目別カバー率 = 供給コマ数 ÷ 需要コマ数 × 100
```

**例**:
- 数学: 1.5 ÷ 3 × 100 = **50%**（不足）
- 英語: 2.5 ÷ 5 × 100 = **50%**（不足）

##### ステップ4: 時間帯の制約も確認する

上記の計算だけでは「講師と生徒の希望時間帯が合っているか」が分からない。

例えば、数学を教えられる講師が月曜しか出勤できないのに、数学を受けたい生徒が全員土曜希望だと、実際にはマッチできない。

そこで、各時間枠ごとに「その科目を教えられる講師がいるか」も確認し、時間帯が合わない場合はカバー率を下げる。

#### 計算式（まとめ）

```
科目別カバー率 = min(供給充足度, 時間帯充足度) × 100

供給充足度 = 按分した供給コマ数 ÷ 需要コマ数

時間帯充足度 =
  全枠の「その科目を教えられる講師数 × 2」の合計 ÷
  全枠の「その科目を受講希望する生徒数」の合計
```

2つの充足度のうち、低い方を採用する。「講師の数が足りない」「時間帯が合わない」のどちらかでも問題があればカバー率は下がる。

#### 表示

| カバー率 | ステータス | 表示色 |
|---------|-----------|--------|
| 100%以上 | 充足 | 緑 |
| 70〜99% | やや不足 | 黄 |
| 70%未満 | 不足 | 赤 |

#### データソース

- 需要: 生徒マスタの「科目別コマ数」
- 供給: 講師マスタの「指導可能科目」「最大コマ/週」
- 時間帯: 講師シフト希望、生徒受講希望

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|

#### カラムマッピング詳細

**講師シフト希望の場合**:

| システム項目 | 説明 | 自動検出キーワード |
|-------------|------|------------------|
| 氏名 | 講師の氏名 | "名前", "氏名", "講師名" |
| 曜日 | 希望曜日 | "曜日" |
| 時限 | 希望時限 | "時限", "コマ", "時間" |
| 希望値 | ○/×/△ | "希望", "可否", "シフト" |

**生徒受講希望の場合**:

| システム項目 | 説明 | 自動検出キーワード |
|-------------|------|------------------|
| 氏名 | 生徒の氏名 | "名前", "氏名", "生徒名" |
| 曜日 | 希望曜日 | "曜日" |
| 時限 | 希望時限 | "時限", "コマ", "時間" |
| 希望値 | ○/×/△ | "希望", "可否" |

**自動検出ロジック**:
1. スプレッドシートのヘッダー行（1行目）を取得
2. 各列名に対してキーワードマッチングを実行
3. マッチした列を自動でマッピング候補として提示
4. 未マッチの列は手動でマッピングを設定

**手動設定UI**:
- ドラッグ&ドロップでシステム項目とスプレッドシート列を対応付け
- プレビュー表示で取り込み結果を事前確認
- マッピング設定の保存・読み込み機能

#### ハード制約（必須・違反不可）

時間割生成において必ず満たさなければならない制約。これらに違反するコマは生成されない。

| ID | 制約名 | 説明 |
|----|--------|------|
| H001 | 科目適合 | 講師の指導可能科目に、生徒の受講科目が含まれていること |
| H002 | 学年適合 | 講師の指導可能学年に、生徒の学年が含まれていること |

## 16. データバージョン管理

### 16.1 バージョン管理が必要なデータ

| データ | 変更頻度 | バージョン管理 |
|--------|---------|--------------|
| 講師マスタ | 中 | **必須** |
| 生徒マスタ | 中 | **必須** |
| 時間割 | 高 | **必須** |
| 希望データ | 中 | 推奨 |
| 制約条件 | 低 | 推奨 |

### 16.2 マスタのバージョン管理

```
Teacher / Student:
  - teacher_id / student_id: ID（不変）
  - version_id: バージョンID（例: T001_v3）
  - version_number: バージョン番号
  - is_current: 現在有効なバージョンか
  - valid_from / valid_to: 有効期間
  - change_reason: 変更理由
```

**更新時の処理**: 新バージョン作成 → 旧バージョンのis_current=false, valid_to=現在日時

### 16.3 時間割バージョン管理

```
Schedule:
  - schedule_id: 時間割ID
  - term_id: タームID（例: 2025-04）
  - version: バージョン番号
  - status: draft / confirmed / archived
  - parent_version: 派生元バージョン
  - master_snapshot: 作成時のマスタバージョン参照
  - generation_config: 生成時の設定（再現性のため）
```

**ステータス遷移**:
- draft: 作成中。削除可能、確定可能
- confirmed: 確定済み。直接編集不可、派生して新バージョン作成
- archived: 過去ターム。参照のみ

**確定済み時間割の保護**:
- 参照・PDF出力: 可能
- 直接編集: 不可
- 振替対応: 新バージョンを作成して反映

## 17. バリデーションルール

### 学年と科目の整合性

| 学年カテゴリ | 設定可能な科目 |
|------------|--------------|
| 小学生（小1〜小6） | 小学英語、公立算数、中受算数、小学国語、小論、公立理科、中受理科、公立社会、中受社会 |
| 中学生（中1〜中3） | 中学英語、公立数学、私立数学、中学国語、中学理科、中学社会 |
| 高校生（高1〜高3） | 高校12英語、高3英語、高校国語、数1A、数ⅡB、数Ⅲ、生物、化学、物理、世界史、日本史、地理 |

### NG関係の相互性

- 講師側でNG生徒を設定 → 生徒側のNG講師に自動追加（逆も同様）
- 片方を削除 → 相互関係も自動削除

### コマ数の整合性

- 講師: 最小コマ ≤ 割り当てコマ ≤ 最大コマ
- 生徒: 科目別コマ数の合計 = 週の受講コマ数
- 連続コマ: 設定値を超えない
