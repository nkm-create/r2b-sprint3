# P008: 条件設定画面 要件定義書 v1

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | P008 |
| 画面名 | 条件設定画面 |
| 対象ユーザー | 教室長 |
| 目的 | マスタデータから自動反映された制約を確認し、ターム固有の調整と全体ポリシーを設定 |

---

## 2. 設計思想：マスタデータとの役割分担

### 2.1 データソースの整理

| データソース | 管理場所 | P008での扱い |
|-------------|---------|--------------|
| 講師の基本情報（科目、コマ数、連続上限等） | P005 講師管理 | **自動反映**（参照のみ） |
| 生徒の基本情報（希望講師、NG講師、性別希望等） | P006 生徒管理 | **自動反映**（参照のみ） |
| 教室設定（ブース数、営業曜日等） | P004 教室設定 | **自動反映**（参照のみ） |
| シフト希望・受講希望 | P007 希望データ | **自動反映**（参照のみ） |
| **ターム固有の追加調整** | P008 条件設定 | **編集可能** |
| **全体ポリシー** | P008 条件設定 | **編集可能** |

### 2.2 P008の役割

1. **マスタから自動反映された制約の確認**: 講師・生徒・教室設定から読み込み、プリセット表示
2. **ターム固有の追加調整**: マスタ設定に対する「このターム限定の上書き/追加」
3. **全体ポリシー設定**: マッチングルール等の全体方針

**ユーザーが再入力する必要がない項目**:
- 講師のコマ数、連続コマ上限 → マスタから自動取得
- 生徒の希望講師、NG講師、性別希望 → マスタから自動取得
- 教室のブース数 → 教室設定から自動取得

## 4. 制約条件の分類

### 4.1 ハード制約（自動適用・変更不可）

**常に適用され、ユーザーが無効化できない制約。**

| ID | 制約名 | データソース |
|----|--------|------------|
| H001 | 科目適合 | 講師マスタ.指導可能科目、生徒マスタ.科目別コマ数 |
| H002 | 学年適合 | 講師マスタ.指導可能学年、生徒マスタ.学年 |
| H003 | 講師出勤可能時間 | 講師シフト希望（○の時間帯のみ） |
| H004 | 生徒通塾可能時間 | 生徒受講希望（○/△の時間帯のみ） |
| H005 | NG組み合わせ禁止 | 講師マスタ.NG生徒ID、生徒マスタ.NG講師ID |

### 4.2 マスタ由来のソフト制約（自動反映・ターム調整可）

**マスタデータから自動的にプリセットされる。ターム固有の調整が可能。**

| ID | 制約名 | データソース | ターム調整の例 |
|----|--------|------------|--------------|
| S001 | 講師コマ数範囲 | 講師マスタ.最小コマ/週、最大コマ/週 | 「今月は田中先生を最大3コマに」 |
| S002 | 講師連続コマ上限 | 講師マスタ.最大連続コマ | 「今月は山本先生を連続2コマまでに」 |
| S003 | 生徒連続コマ上限 | 生徒マスタ.最大連続コマ | 「今月は鈴木くんを連続1コマまでに」 |
| S004 | 講師希望 | 生徒マスタ.希望講師ID | 「今月は追加で佐藤先生を希望」 |
| S005 | 講師NG | 生徒マスタ.NG講師ID | 「今月は追加で田中先生をNG」 |
| S006 | 講師性別希望 | 生徒マスタ.講師希望性別 | （通常は調整不要） |
| S007 | ブースキャパシティ | P004.ブース数 | 「今月は改装で最大6コマに」 |

### 4.3 全体ポリシー（P008で設定）

**マッチング・最適化の全体方針。教室全体に適用。**

| ID | ポリシー名 | 説明 |
|----|-----------|------|
| P001 | 志望レベル→大学ランクマッピング | 生徒の志望レベルに応じて講師ランクを制限 |
| P002 | 中受経験優先 | 中受科目に中受経験講師を優先配置 |
| P003 | 高受経験優先 | 高受科目に高受経験講師を優先配置 |
| P004 | 同一科目ペアリング | 1対2で同一科目の生徒同士を優先 |
| P005 | 充足率目標 | 1対2コマの充足率目標値 |
| P006 | 時間帯優先 | 特定時間帯を優先的に埋める |

### 4.4 ターム固有の追加制約（P008で設定）

**マスタにはない、このタームだけの特別な制約。**

| ID | 制約名 | 用途 |
|----|--------|------|
| T001 | 講師科目限定 | 「このタームは佐藤先生を中受算数のみに」 |
| T002 | 講師曜日限定 | 「このタームは田中先生を月水金のみに」 |

## 5. 機能詳細

### F070: ターム選択・作成

タームを選択または新規作成。

#### 入力項目

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|
| ターム名 | string | ○ | 例: "2025年4月期" |
| 開始日 | date | ○ | ターム開始日 |
| 終了日 | date | ○ | ターム終了日 |

#### 処理

1. 新規作成時: マスタデータを読み込み、ソフト制約をプリセット
2. 既存選択時: 保存済みの調整設定を読み込み

#### 各ポリシーの詳細

##### P001: 志望レベル→大学ランクマッピング

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 有効/無効 | トグルスイッチ | ON / OFF | - |
| 志望レベルA許容ランク | チェックボックス | A / B / C | 1件以上 |
| 志望レベルB許容ランク | チェックボックス | A / B / C | 1件以上 |
| 志望レベルC許容ランク | チェックボックス | A / B / C | 1件以上 |

**データソース**: 生徒マスタ.志望レベル、講師マスタ.大学ランク

##### P002: 中受経験優先

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 有効/無効 | トグルスイッチ | ON / OFF | - |
| 対象科目 | チェックボックス | 中受算数 / 中受理科 / 中受社会 | 1件以上（有効時） |

**データソース**: 講師マスタ.中学受験経験（○がある講師を優先）

##### P003: 高受経験優先

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 有効/無効 | トグルスイッチ | ON / OFF | - |
| 対象科目 | チェックボックス | 中学英語/公立数学/私立数学/中学国語/中学理科/中学社会 | 1件以上（有効時） |

**データソース**: 講師マスタ.高校受験経験（○がある講師を優先）

##### P004: 同一科目ペアリング

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 有効/無効 | トグルスイッチ | ON / OFF | - |

##### P005: 充足率目標

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 目標値 | スライダー + 数値表示 | 50% 〜 100%（5%刻み） | 必須 |

##### P006: 時間帯優先

| パラメータ | 入力形式 | 選択肢/範囲 | バリデーション |
|-----------|---------|------------|--------------|
| 有効/無効 | トグルスイッチ | ON / OFF | - |
| 時間帯 | マトリクス選択（曜日×時限） | 営業曜日 × 設定済み時限 | 1件以上（有効時） |

### F074: 条件の保存・読み込み

設定したターム調整・ポリシーをテンプレートとして保存・再利用する。

#### 機能

| 操作 | 説明 |
|------|------|
| テンプレート保存 | 現在の全体ポリシー設定をテンプレートとして保存 |
| テンプレート読み込み | 保存済みテンプレートを新しいタームに適用 |
| 前タームからコピー | 前タームの調整設定をコピー |

## 9. 設計根拠

### 9.1 マスタとの役割分担

| 観点 | マスタ管理画面 | P008条件設定 |
|------|--------------|-------------|
| 目的 | 恒久的な基本設定 | ターム限定の調整 |
| 変更頻度 | 低（講師入退職、生徒入退塾時） | 高（毎ターム） |
| 影響範囲 | 全ターム | 当該タームのみ |

### 9.2 ユーザー負荷の軽減

- **再入力不要**: マスタデータが自動でプリセットされる
- **差分設定**: 変更がある部分だけを設定
- **テンプレート**: よく使う設定パターンを再利用可能

### 9.3 ターム調整の必要性

| ユースケース | 対応方法 |
|------------|---------|
| 「今月は田中先生が試験期間なのでコマ数を減らしたい」 | S001でコマ数上書き |
| 「鈴木くんが今月から佐藤先生を希望している」 | S004で講師希望追加 |
| 「改装工事で一時的にブース数が減る」 | S007でキャパシティ上書き |
