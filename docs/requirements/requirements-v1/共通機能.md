# 共通機能 要件定義書 v1

## 1. 概要

本ドキュメントは、全画面で共通して使用される機能の要件を定義する。

---

## 2. 検索・フィルター（全一覧画面共通）

### 2.1 テキスト検索

| 項目 | 仕様 |
|------|------|
| 検索対象 | 名前での部分一致 |
| 検索タイミング | 入力確定時（Enter）またはデバウンス（300ms） |
| 最小文字数 | 1文字以上 |
| 大文字/小文字 | 区別しない |

### 2.2 フィルター

| 項目 | 仕様 |
|------|------|
| 複数条件 | AND条件で絞り込み |
| フィルタークリア | 「リセット」ボタンで全解除 |
| フィルター保存 | ブラウザのローカルストレージに保存（任意） |

### 2.3 ソート

| 項目 | 仕様 |
|------|------|
| 対象 | 各列での昇順/降順 |
| デフォルト | 作成日時の降順 |
| 複数列ソート | 非対応 |

## 3. ページネーション

### 3.1 仕様

| 項目 | 仕様 |
|------|------|
| 1ページあたり件数 | 20件（デフォルト） |
| 選択可能件数 | 10 / 20 / 50 / 100件 |
| 表示形式 | 「1-20 / 92件」形式 |
| ページ移動 | 前へ/次へ、ページ番号直接指定 |

### 3.2 URL同期

- ページ番号、件数をURLクエリパラメータに反映
- ブックマーク・共有が可能

## 4. 削除処理

### 4.1 論理削除

| 項目 | 仕様 |
|------|------|
| 方式 | deleted_atフラグによる論理削除 |
| 削除日時 | deleted_atにタイムスタンプを記録 |
| 一覧表示 | 削除済みデータは非表示 |

### 4.2 復元

| 項目 | 仕様 |
|------|------|
| 復元権限 | システム管理者のみ |
| 復元操作 | deleted_atをnullに設定 |
| 復元履歴 | 監査ログに記録 |

## 5. CSVエクスポート

### 5.1 対応画面

| 対象画面 | エクスポート内容 | ファイル名形式 |
|---------|----------------|---------------|
| P005 講師管理 | 講師一覧（全件またはフィルター適用後） | teachers_YYYYMMDD.csv |
| P006 生徒管理 | 生徒一覧（全件またはフィルター適用後） | students_YYYYMMDD.csv |
| P009 時間割作成 | 時間割データ（全体/講師別/生徒別） | schedule_{term_id}_YYYYMMDD.csv |

### 5.2 エクスポート処理

1. エクスポートボタンをクリック
2. エクスポート範囲を選択（全件/表示中のみ）
3. CSVファイルを生成・ダウンロード

### 5.3 CSV形式

| 項目 | 仕様 |
|------|------|
| 文字コード | UTF-8（BOM付き、Excel対応） |
| 区切り文字 | カンマ |
| ヘッダー行 | あり |
| 日付形式 | YYYY-MM-DD |
| 改行 | CRLF |
| 引用符 | ダブルクォート（値にカンマ・改行を含む場合） |

## 6. データバージョン管理

### 6.1 バージョン管理が必要なデータ

| データ | 変更頻度 | バージョン管理 |
|--------|---------|--------------|
| 講師マスタ | 中 | **必須** |
| 生徒マスタ | 中 | **必須** |
| 時間割 | 高 | **必須** |
| 希望データ | 中 | 推奨 |
| 制約条件 | 低 | 推奨 |

### 6.2 マスタのバージョン管理

```
Teacher / Student:
  - teacher_id / student_id: ID（不変）
  - version_id: バージョンID（例: T001_v3）
  - version_number: バージョン番号
  - is_current: 現在有効なバージョンか
  - valid_from / valid_to: 有効期間
  - change_reason_type: 変更理由種別（enum）
  - change_reason_note: 変更理由補足（自由入力、任意）
```

#### 変更理由種別（change_reason_type）

**講師マスタの場合**:

| 値 | 説明 |
|-----|------|
| SCHEDULE_CHANGE | シフト・コマ数の変更 |
| SUBJECT_CHANGE | 指導可能科目の変更 |
| GRADE_CHANGE | 指導可能学年の変更 |
| NG_CHANGE | NG生徒の追加・削除 |
| STATUS_CHANGE | ステータス変更（有効/無効） |
| INITIAL | 初回登録 |
| OTHER | その他 |

**生徒マスタの場合**:

| 値 | 説明 |
|-----|------|
| COURSE_CHANGE | 受講科目・コマ数の変更 |
| GRADE_UP | 学年進級 |
| PREFERENCE_CHANGE | 希望講師・NG講師の変更 |
| GOAL_CHANGE | 志望レベル・通塾目的の変更 |
| STATUS_CHANGE | ステータス変更（有効/無効） |
| INITIAL | 初回登録 |
| OTHER | その他 |

※ `OTHER` を選択した場合は `change_reason_note` の入力を必須とする。

**更新時の処理**: 新バージョン作成 → 旧バージョンのis_current=false, valid_to=現在日時

### 6.3 時間割バージョン管理

```
Schedule:
  - schedule_id: 時間割ID
  - term_id: タームID（例: 2025-04）
  - version: バージョン番号
  - status: draft / confirmed / archived
  - parent_version: 派生元バージョン
  - master_snapshot: 作成時のマスタバージョン参照
  - generation_config: 生成時の設定（再現性のため）
```

**ステータス遷移**:
- draft: 作成中。削除可能、確定可能
- confirmed: 確定済み。直接編集不可、派生して新バージョン作成
- archived: 過去ターム。参照のみ

**確定済み時間割の保護**:
- 参照・PDF出力: 可能
- 直接編集: 不可
- 振替対応: 新バージョンを作成して反映

## 8. バリデーションルール

### 8.1 学年と科目の整合性

| 学年カテゴリ | 設定可能な科目 |
|------------|--------------|
| 小学生（小1〜小6） | 小学英語、公立算数、中受算数、小学国語、小論、公立理科、中受理科、公立社会、中受社会 |
| 中学生（中1〜中3） | 中学英語、公立数学、私立数学、中学国語、中学理科、中学社会 |
| 高校生（高1〜高3） | 高校12英語、高3英語、高校国語、数1A、数ⅡB、数Ⅲ、生物、化学、物理、世界史、日本史、地理 |

### 8.2 NG関係の相互性

- 講師側でNG生徒を設定 → 生徒側のNG講師に自動追加（逆も同様）
- 片方を削除 → 相互関係も自動削除

### 8.3 コマ数の整合性

- 講師: 最小コマ ≤ 割り当てコマ ≤ 最大コマ
- 生徒: 科目別コマ数の合計 = 週の受講コマ数
- 連続コマ: 設定値を超えない

## 9. エラーハンドリング

### 9.1 エラー分類

| 分類 | 説明 | 例 |
|------|------|-----|
| バリデーションエラー | 入力値の形式・範囲エラー | 必須項目未入力、形式不正 |
| ビジネスエラー | 業務ルール違反 | 重複登録、整合性エラー |
| システムエラー | サーバー・通信エラー | タイムアウト、500エラー |

### 9.2 エラー表示

| エラー種別 | 表示方法 |
|-----------|---------|
| フィールドエラー | 該当フィールドの下に赤字表示 |
| フォームエラー | フォーム上部にサマリー表示 |
| システムエラー | トースト通知 |
