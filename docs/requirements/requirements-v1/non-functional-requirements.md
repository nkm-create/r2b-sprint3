# 非機能要件書

## 1. 概要

本ドキュメントは、学習塾時間割最適化システムの非機能要件を定義する。

---

## 2. 性能要件

### 2.1 レスポンスタイム


| カテゴリ   | 操作            | 目標値    | 許容最大値 |
| ------ | ------------- | ------ | ----- |
| 画面表示   | ダッシュボード初期表示   | 1秒以内   | 3秒    |
| 画面表示   | 一覧画面表示（20件）   | 0.5秒以内 | 2秒    |
| CRUD操作 | 登録・更新・削除      | 0.5秒以内 | 2秒    |
| 検索     | テキスト検索・フィルター  | 0.5秒以内 | 2秒    |
| 時間割生成  | 小規模（講師15名以下）  | 30秒以内  | 60秒   |
| 時間割生成  | 中規模（講師16〜35名） | 2分以内   | 5分    |
| 時間割生成  | 大規模（講師36〜50名） | 5分以内   | 10分   |
| 外部連携   | Google Form同期 | 5秒以内   | 15秒   |
| PDF出力  | 時間割PDF生成      | 3秒以内   | 10秒   |

## 4. セキュリティ要件

### 4.1 認証・認可


| 項目             | 仕様                  |
| -------------- | ------------------- |
| 認証方式           | JWT（JSON Web Token） |
| アクセストークン有効期限   | 1時間                 |
| リフレッシュトークン有効期限 | 7日間                 |
| パスワード要件        | 8文字以上、英数字混合         |
| パスワードハッシュ      | bcrypt（コスト係数12）     |
| ログイン失敗ロック      | 5回失敗で30分ロック         |
| セッション管理        | サーバーサイドでのトークン無効化対応  |


### 4.2 通信セキュリティ


| 項目    | 仕様                           |
| ----- | ---------------------------- |
| プロトコル | HTTPS（TLS 1.3）               |
| 証明書   | Let's Encrypt または同等の信頼性を持つCA |
| HSTS  | 有効（max-age=31536000）         |
| 内部通信  | VPC内プライベート通信                 |


### 4.3 データ保護


| 項目          | 仕様                 |
| ----------- | ------------------ |
| データベース暗号化   | 保存時暗号化（AES-256）    |
| PII（個人識別情報） | 氏名、メールアドレス、電話番号を含む |
| PIIアクセスログ   | 全アクセスを記録、90日間保持    |
| データマスキング    | 開発・テスト環境では氏名をマスク   |


### 4.4 脆弱性対策


| 脆弱性         | 対策               |
| ----------- | ---------------- |
| SQLインジェクション | パラメータ化クエリの使用     |
| XSS         | 出力エスケープ、CSP設定    |
| CSRF        | CSRFトークンの検証      |
| 認証バイパス      | 全APIでの認証チェック     |
| 不正なリダイレクト   | ホワイトリスト方式のリダイレクト |
| ファイルアップロード  | ファイルタイプ検証、サイズ制限  |


### 4.5 監査ログ


| 項目    | 仕様                           |
| ----- | ---------------------------- |
| 記録対象  | ログイン/ログアウト、データ変更、設定変更、エクスポート |
| 記録内容  | 日時、ユーザーID、操作内容、対象データ、IPアドレス  |
| 保持期間  | 1年間                          |
| 改ざん防止 | 書き込み専用ストレージ                  |

## 5. 信頼性要件

### 5.1 データ整合性


| 項目       | 仕様                 |
| -------- | ------------------ |
| トランザクション | ACID準拠             |
| 同時更新制御   | 楽観的ロック（バージョン番号による） |
| 外部キー制約   | 厳密に適用              |
| バリデーション  | サーバーサイドで必ず実行       |


### 5.2 エラーハンドリング


| 項目       | 仕様                          |
| -------- | --------------------------- |
| エラー分類    | ビジネスエラー / システムエラー / 外部連携エラー |
| エラーレスポンス | 統一されたJSON形式                 |
| エラーログ    | 全エラーをログ記録                   |
| リトライ     | 外部API呼び出しは3回までリトライ          |


### 5.3 フェイルセーフ


| 項目              | 仕様             |
| --------------- | -------------- |
| 時間割生成失敗         | 中間結果を保存、再開可能   |
| Google Form連携失敗 | 前回同期データを維持     |
| PDF生成失敗         | エラー通知、再試行ボタン表示 |
