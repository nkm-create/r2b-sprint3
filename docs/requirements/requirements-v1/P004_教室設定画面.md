# P004: 教室設定画面 要件定義書 v1

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | P004 |
| 画面名 | 教室設定画面 |
| 対象ユーザー | 教室長 |
| 目的 | 教室の基本設定（時間枠、営業曜日、キャパシティ、外部連携） |

---

|----|--------|--------|
| F030 | 時間枠設定 | Must |

| F034 | Google Form連携設定 | Must |

---

## 3. 機能詳細

### F030: 時間枠設定

#### 入力項目

| 項目名 | 型 | 説明 |
|--------|-----|------|
| 限の数 | number | 1日あたりの時限数（平日: 4限、土曜: 5限） |
| 各限の時間 | object[] | 開始時刻〜終了時刻 |

#### 処理

- 時間枠の追加/編集/削除
- 曜日ごとの時間枠カスタマイズ

#### 出力

- 設定済み時間枠の一覧表示

---

### F031: 営業曜日設定

#### 入力項目

| 項目名 | 型 | 説明 |
|--------|-----|------|
| 営業曜日 | boolean[] | 月〜土の各曜日の営業有無 |

#### 処理

月〜土の中から営業日を選択（日曜は固定休み）

---

### F032: キャパシティ設定

#### 入力項目

| 項目名 | 型 | 説明 |
|--------|-----|------|
| ブース数 | number | 同時実施可能なコマ数の上限 |

#### 処理

- ブース数の設定
- 時間枠ごとのキャパシティオーバーライド（任意）

---

### F033: デフォルト設定適用

#### 処理

標準的な教室設定テンプレートを適用

#### テンプレート内容

- 平日: 4限（16:00-17:20, 17:30-18:50, 19:00-20:20, 20:30-21:50）
- 土曜: 5限（13:00-14:20, 14:30-15:50, 16:00-17:20, 17:30-18:50, 19:00-20:20）
- 営業日: 月〜土
- ブース数: 10

---

### F034: Google Form連携設定

#### 入力項目

| 項目名 | 型 | 必須 | 説明 |
|--------|-----|------|------|
| データ種別 | enum | ○ | "講師シフト希望" / "生徒受講希望" |
| スプレッドシートURL | string | ○ | Google SheetsのURL |
| シート名 | string | ○ | 対象シート名 |
| カラムマッピング | object | ○ | 各列と項目の対応付け |

#### 処理

OAuth認証 → シート接続確認 → カラムマッピング（自動検出/手動設定）→ 設定保存

#### カラムマッピング詳細

**講師シフト希望の場合**:

| システム項目 | 説明 | 自動検出キーワード |
|-------------|------|------------------|
| 氏名 | 講師の氏名 | "名前", "氏名", "講師名" |
| 曜日 | 希望曜日 | "曜日" |
| 時限 | 希望時限 | "時限", "コマ", "時間" |
| 希望値 | ○/×/△ | "希望", "可否", "シフト" |

**生徒受講希望の場合**:

| システム項目 | 説明 | 自動検出キーワード |
|-------------|------|------------------|
| 氏名 | 生徒の氏名 | "名前", "氏名", "生徒名" |
| 曜日 | 希望曜日 | "曜日" |
| 時限 | 希望時限 | "時限", "コマ", "時間" |
| 希望値 | ○/×/△ | "希望", "可否" |

**自動検出ロジック**:
1. スプレッドシートのヘッダー行（1行目）を取得
2. 各列名に対してキーワードマッチングを実行
3. マッチした列を自動でマッピング候補として提示
4. 未マッチの列は手動でマッピングを設定

**手動設定UI**:
- ドラッグ&ドロップでシステム項目とスプレッドシート列を対応付け
- プレビュー表示で取り込み結果を事前確認
- マッピング設定の保存・読み込み機能

#### Google API技術仕様

| 項目 | 仕様 |
|------|------|
| 使用API | Google Sheets API v4 |
| 認証方式 | OAuth 2.0 |
| 必要スコープ | `https://www.googleapis.com/auth/spreadsheets.readonly` |
| トークン有効期限 | アクセストークン: 1時間、リフレッシュトークン: 無期限（取り消しまで） |
| APIレート制限対応 | 指数バックオフによるリトライ（最大3回） |

#### 同期タイミング

| トリガー | 説明 |
|---------|------|
| 手動同期 | P007画面の「Google Form同期」ボタン押下時 |
| ターム作成時 | 新規ターム作成時に自動同期を実行 |

※定期自動同期は実装しない。教室長が明示的に同期を実行する運用とする。

#### エラーハンドリング

| エラー種別 | 対応 |
|-----------|------|
| 認証エラー（トークン失効） | 再認証を促すダイアログ表示 |
| スプレッドシート未検出 | 「スプレッドシートが見つかりません。URLを確認してください」 |
| シート未検出 | 「指定されたシートが見つかりません」 |
| 権限エラー | 「スプレッドシートへのアクセス権限がありません」 |
| レート制限 | 「一時的にアクセスできません。しばらく待ってから再試行してください」 |
| マッピング不整合 | 取り込み失敗行をスキップし、エラーレポートを表示 |

## 4. UI状態遷移

```
[教室設定画面]
    ├─→ 時間枠タブ → [時間枠設定]
    │                   ├─→ 追加 → [時間枠追加モーダル]
    │                   ├─→ 編集 → [時間枠編集モーダル]
    │                   └─→ 削除 → [確認ダイアログ] → 削除実行
    ├─→ 営業曜日タブ → [営業曜日設定]
    ├─→ キャパシティタブ → [キャパシティ設定]
    ├─→ 連携タブ → [Google Form連携設定]
    │               ├─→ 追加 → [連携設定モーダル]
    │               │           ├─→ OAuth認証 → [Google認証画面]
    │               │           └─→ マッピング設定 → [カラムマッピング画面]
    │               └─→ 編集/削除 → [各操作]
    └─→ デフォルト適用 → [確認ダイアログ] → 設定適用
```
