# IPO一覧（Input / Process / Output）

## 1. 概要

本ドキュメントは、学習塾時間割最適化システムの全機能について、入力・処理・出力を整理したものである。

### 1.1 参照ドキュメント

| ドキュメント | 内容 |
|------------|------|
| functional-requirements.md | 機能要件書 |
| requirements-v1/*.md | 各画面の要件定義書 |
| operations/operations-policy.md | 運用方針定義書 |

### 1.2 凡例

| 項目 | 説明 |
|------|------|
| Input | 機能の実行に必要な入力データ |
| Process | 入力データに対して行う処理 |
| Output | 処理結果として出力されるデータ |

---

## 2. 認証・認可機能（P001）

### F001: ログイン

| 区分 | 内容 |
|------|------|
| **Input** | メールアドレス（string）、パスワード（string） |
| **Process** | 1. メールアドレスでユーザー検索<br>2. パスワードハッシュ比較<br>3. JWT（アクセストークン・リフレッシュトークン）発行<br>4. ログイン試行回数管理 |
| **Output** | 成功: アクセストークン、リフレッシュトークン、役割別リダイレクト先URL<br>失敗: エラーメッセージ |

### F002: パスワードリセット

| 区分 | 内容 |
|------|------|
| **Input** | メールアドレス（string） |
| **Process** | 1. メールアドレス存在確認<br>2. リセットトークン生成（有効期限24時間）<br>3. リセットメール送信 |
| **Output** | 成功: メール送信完了メッセージ<br>失敗: エラーメッセージ |

### F003: ログイン状態維持

| 区分 | 内容 |
|------|------|
| **Input** | リフレッシュトークン（string） |
| **Process** | 1. リフレッシュトークン検証<br>2. 新アクセストークン発行 |
| **Output** | 成功: 新アクセストークン<br>失敗: ログイン画面へリダイレクト |

## 3. ダッシュボード機能（P002）

### F010: ダッシュボード表示（教室長）

| 区分 | 内容 |
|------|------|
| **Input** | ユーザーID、教室ID |
| **Process** | 1. ユーザー権限確認<br>2. 教室データ取得<br>3. 各ウィジェットデータを並列取得 |
| **Output** | ダッシュボードデータ（充足率サマリー、ヒートマップ、科目別カバー率、需給バランス、通知） |

### F011: 充足率サマリー表示

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID、タームID |
| **Process** | 1. 時間割データ取得<br>2. 1対2コマ数・1対1コマ数を集計<br>3. 充足率計算: (1対2コマ数 × 2) / (1対2コマ数 × 2 + 1対1コマ数) × 100<br>4. 前週・前月のデータ取得・比較計算 |
| **Output** | 充足率（%）、1対2コマ数、1対1コマ数、未配置コマ数、前週比（増減）、前月比（増減） |

### F012: 人員状況ヒートマップ表示

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID、タームID |
| **Process** | 1. 講師シフト希望取得（供給枠）<br>2. 生徒受講希望取得（需要数）<br>3. 曜日×時限ごとに需給差計算: 供給枠 - 需要人数 |
| **Output** | 曜日×時限マトリクス（各セル: 需給差、ステータス色） |

### F013: 科目別カバー率表示

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID、タームID |
| **Process** | 1. 生徒の科目別コマ数を集計（需要）<br>2. 講師の最大コマ数を科目別に按分（供給）<br>3. 時間帯充足度を計算<br>4. カバー率 = min(供給充足度, 時間帯充足度) × 100 |
| **Output** | 科目別カバー率リスト（科目名、カバー率、ステータス色） |

### F014: 需給バランス表示

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID、タームID |
| **Process** | 1. 講師供給コマ数の週合計を計算<br>2. 生徒需要コマ数の週合計を計算<br>3. バランス比率を算出 |
| **Output** | 週間供給コマ数、週間需要コマ数、バランスグラフデータ |

### F015: 通知・アラート表示

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID |
| **Process** | 1. 充足率閾値チェック<br>2. 未回答希望データ検索<br>3. ターム確定期限チェック<br>4. 未読/既読状態管理 |
| **Output** | 通知リスト（種別、メッセージ、重要度、リンク先、未読/既読状態）、重要度でソート |

## 4. マネジメントダッシュボード機能（P003）

### F020: マネジメントダッシュボード表示

| 区分 | 内容 |
|------|------|
| **Input** | ユーザーID（エリアマネジャー） |
| **Process** | 1. 担当エリア取得<br>2. エリア内全教室の統計を集計 |
| **Output** | エリアサマリー（教室数、平均充足率）、教室別充足率一覧 |

### F021: 教室別充足率一覧表示

| 区分 | 内容 |
|------|------|
| **Input** | エリアID（ログインユーザーの担当エリアから取得）、ソート条件、フィルター条件 |
| **Process** | 1. エリア内教室一覧取得<br>2. 各教室の充足率・前月比を計算<br>3. ソート・フィルター適用 |
| **Output** | 教室リスト（教室名、充足率、ステータス、前月比、講師数、生徒数、時間割ステータス、アラートバッジ） |

### F022: 教室詳細ドリルダウン

| 区分 | 内容 |
|------|------|
| **Input** | 教室ID |
| **Process** | 1. 教室の詳細データ取得<br>2. P002相当のダッシュボードデータを構築（参照モード） |
| **Output** | 教室詳細（サマリー: 充足率/講師数/生徒数、充足率推移グラフ（過去6ターム）、人員ヒートマップ、時間割サマリー、問題点リスト） |

### F023: 問題教室アラート

| 区分 | 内容 |
|------|------|
| **Input** | エリアID |
| **Process** | 1. 各教室の充足率チェック<br>  - 60%未満 → 危険<br>  - 60〜79% → 警告<br>2. 前月比チェック（10pt以上低下 → 危険）<br>3. 確定期限チェック（未確定で期限超過 → 警告） |
| **Output** | アラート付き教室リスト（教室名、アラートレベル（危険/警告）、原因） |

### F024: 教室比較

| 区分 | 内容 |
|------|------|
| **Input** | 教室IDリスト（2〜4件） |
| **Process** | 1. 各教室の統計データ取得<br>2. 比較用に横並びでデータ整形<br>3. 推移データ取得（グラフ用） |
| **Output** | 比較データ（充足率、前月比、講師数/生徒数、推移グラフデータ） |

| 区分 | 内容 |
|------|------|

| **Process** | 【Schedule Orchestrator（状態マシン）】<br>1. Evaluatorでスコア計算<br>2. Advisorで改善提案生成（LLM）<br>3. ユーザー判断（対話モード）または自動選択<br>4. Builderで改善適用<br>5. 終了条件判定:<br>  - 目標達成（充足率≥85%）<br>  - 改善余地なし（2回連続で改善幅<1%）<br>  - 反復上限（10回）<br>  - ユーザー確定<br>ループ |
