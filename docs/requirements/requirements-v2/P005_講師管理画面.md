## 3. DBマッピング

### 3.1 関連テーブル

| テーブル | 役割 |
|---------|------|
| teachers | 講師マスタ（バージョン管理） |

| teacher_grades | 講師-指導可能学年（中間） |
| ng_relations | NG関係 |
| subjects | 科目マスタ（参照のみ） |

### 3.2 カラムマッピング

**teachers テーブル**:

| 画面項目 | DBカラム | 型 | 制約 |
|---------|---------|-----|------|
| 氏名 | name | VARCHAR(50) | NN |
| 性別 | gender | gender | NN |
| 最小コマ/週 | min_slots_per_week | INTEGER | NN |
| 最大コマ/週 | max_slots_per_week | INTEGER | NN |
| 最大連続コマ | max_consecutive_slots | INTEGER | NN, 1〜4 |
| 中学受験経験 | has_jhs_exam_experience | BOOLEAN | DEFAULT false |
| 高校受験経験 | has_hs_exam_experience | BOOLEAN | DEFAULT false |
| 大学ランク | university_rank | university_rank | A / B / C / null |
| ステータス | status | teacher_status | active / inactive |

**バージョン管理カラム**:

| カラム | 用途 |
|-------|------|
| teacher_version_id | バージョンごとのPK（UUID） |
| teacher_id | 不変ID（VARCHAR(20)、例: T001） |
| classroom_id | 教室ID（UUID） |
| version_number | バージョン番号（INTEGER） |
| is_current | 現行フラグ（BOOLEAN） |
| valid_from | 有効開始日時 |
| valid_to | 有効終了日時 |
| change_reason_type | 変更理由種別 |
| change_reason_note | 変更理由補足 |

## 4. 機能詳細

### F040: 講師一覧表示

#### 出力項目

| 項目名 | DBカラム | 説明 |
|--------|---------|------|
| 氏名 | teachers.name | 講師の氏名 |
| 性別 | teachers.gender | male→男 / female→女 |
| 指導可能科目 | teacher_subjects.subject_id → subjects.subject_name | 科目一覧（省略表示） |
| 指導可能学年 | teacher_grades.grade | 学年範囲 |
| 最小/最大コマ | teachers.min_slots_per_week / max_slots_per_week | 週あたりのコマ数 |
| ステータス | teachers.status | active→有効 / inactive→無効 |

#### API仕様

```
GET /api/classrooms/{classroom_id}/teachers

クエリパラメータ:
- page: ページ番号（デフォルト: 1）
- per_page: 件数（デフォルト: 20）
- q: 検索文字列（氏名部分一致）
- filter[status]: active / inactive / all
- filter[gender]: male / female / all
- filter[subject_id]: 科目IDで絞り込み
- filter[grade_category]: elementary / junior_high / high_school
- filter[university_rank]: A / B / C
- sort: name / -created_at など

レスポンス:
{
  "data": [
    {
      "teacher_id": "T001",
      "teacher_version_id": "uuid",
      "version_number": 3,
      "name": "山田太郎",
      "gender": "male",
      "subjects": [
        {"subject_id": "JHS_MATH_PUB", "subject_name": "公立数学"},
        {"subject_id": "JHS_MATH_PRI", "subject_name": "私立数学"}
      ],
      "grades": ["jhs1", "jhs2", "jhs3", "hs1", "hs2"],
      "min_slots_per_week": 4,
      "max_slots_per_week": 8,
      "max_consecutive_slots": 3,
      "has_jhs_exam_experience": false,
      "has_hs_exam_experience": true,
      "university_rank": "A",
      "status": "active"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total_count": 25,
    "total_pages": 2
  }
}
```

#### SQL例

```sql
SELECT
  t.teacher_id,
  t.teacher_version_id,
  t.version_number,
  t.name,
  t.gender,
  t.min_slots_per_week,
  t.max_slots_per_week,
  t.max_consecutive_slots,
  t.has_jhs_exam_experience,
  t.has_hs_exam_experience,
  t.university_rank,
  t.status,
  array_agg(DISTINCT ts.subject_id) as subject_ids,
  array_agg(DISTINCT tg.grade) as grades
FROM teachers t
LEFT JOIN teacher_subjects ts ON t.teacher_version_id = ts.teacher_version_id
LEFT JOIN teacher_grades tg ON t.teacher_version_id = tg.teacher_version_id
WHERE t.classroom_id = :classroom_id
  AND t.is_current = true
  AND t.deleted_at IS NULL
GROUP BY t.teacher_version_id
ORDER BY t.created_at DESC
LIMIT :per_page OFFSET :offset;
```

### F041: 講師新規登録

#### 入力項目

| 項目名 | 型 | 必須 | DBカラム | バリデーション |
|--------|-----|------|---------|--------------|
| 氏名 | string | ○ | name | 最大50文字 |
| 性別 | enum | ○ | gender | male / female |
| 指導可能科目 | string[] | ○ | teacher_subjects | 科目マスタから選択、1件以上 |
| 指導可能学年 | string[] | ○ | teacher_grades | ele1〜hs3から選択、1件以上 |
| 最小コマ/週 | number | ○ | min_slots_per_week | 1以上、最大コマ以下 |
| 最大コマ/週 | number | ○ | max_slots_per_week | 最小コマ以上 |
| 最大連続コマ | number | ○ | max_consecutive_slots | 1〜4 |
| NG生徒 | string[] | - | ng_relations | 生徒マスタから選択 |
| 中学受験経験 | boolean | - | has_jhs_exam_experience | デフォルト: false |
| 高校受験経験 | boolean | - | has_hs_exam_experience | デフォルト: false |
| 大学ランク | enum | - | university_rank | A / B / C / null |

#### API仕様

```
POST /api/classrooms/{classroom_id}/teachers

リクエスト:
{
  "name": "山田太郎",
  "gender": "male",
  "subject_ids": ["JHS_MATH_PUB", "JHS_MATH_PRI"],
  "grades": ["jhs1", "jhs2", "jhs3"],
  "min_slots_per_week": 4,
  "max_slots_per_week": 8,
  "max_consecutive_slots": 3,
  "ng_student_ids": [],
  "has_jhs_exam_experience": false,
  "has_hs_exam_experience": true,
  "university_rank": "A"
}

成功レスポンス (201 Created):
{
  "teacher_id": "T001",
  "teacher_version_id": "uuid",
  "version_number": 1,
  "message": "講師を登録しました"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 1. teacher_id採番（教室内でユニーク）
SELECT COALESCE(MAX(CAST(SUBSTRING(teacher_id FROM 2) AS INTEGER)), 0) + 1 AS next_num
FROM teachers WHERE classroom_id = :classroom_id;
-- → teacher_id = 'T' || LPAD(next_num, 3, '0')

-- 2. teachersに新規バージョン作成
INSERT INTO teachers (
  teacher_version_id, teacher_id, classroom_id, version_number,
  is_current, valid_from, change_reason_type,
  name, gender, min_slots_per_week, max_slots_per_week,
  max_consecutive_slots, has_jhs_exam_experience, has_hs_exam_experience,
  university_rank, status
) VALUES (
  gen_random_uuid(), :teacher_id, :classroom_id, 1,
  true, NOW(), 'INITIAL',
  :name, :gender, :min_slots_per_week, :max_slots_per_week,
  :max_consecutive_slots, :has_jhs_exam_experience, :has_hs_exam_experience,
  :university_rank, 'active'
) RETURNING teacher_version_id;

-- 3. teacher_subjectsに指導可能科目を登録
INSERT INTO teacher_subjects (teacher_version_id, subject_id)
SELECT :teacher_version_id, unnest(:subject_ids::varchar[]);

-- 4. teacher_gradesに指導可能学年を登録
INSERT INTO teacher_grades (teacher_version_id, grade)
SELECT :teacher_version_id, unnest(:grades::grade[]);

-- 5. NG関係を登録（ng_student_idsがある場合）
INSERT INTO ng_relations (ng_relation_id, teacher_id, student_id, created_by)
SELECT gen_random_uuid(), :teacher_id, unnest(:ng_student_ids::varchar[]), 'teacher';

-- 6. 監査ログ記録
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Teacher', :teacher_id, :new_value_json);

COMMIT;
```

### F042: 講師情報編集

F041と同じ入力項目。変更時は新バージョンを作成。

#### API仕様

```
PUT /api/classrooms/{classroom_id}/teachers/{teacher_id}

リクエスト:
{
  "name": "山田太郎",
  "gender": "male",
  "subject_ids": ["JHS_MATH_PUB", "JHS_MATH_PRI", "HS_MATH_1A"],
  "grades": ["jhs1", "jhs2", "jhs3", "hs1"],
  "min_slots_per_week": 4,
  "max_slots_per_week": 10,
  "max_consecutive_slots": 3,
  "ng_student_ids": ["S005"],
  "has_jhs_exam_experience": false,
  "has_hs_exam_experience": true,
  "university_rank": "A",
  "change_reason_type": "SCHEDULE_CHANGE",
  "change_reason_note": "最大コマ数を増加"
}

成功レスポンス (200 OK):
{
  "teacher_id": "T001",
  "teacher_version_id": "new-uuid",
  "version_number": 4,
  "message": "講師情報を更新しました"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 1. 現在のバージョンを取得
SELECT teacher_version_id, version_number
FROM teachers
WHERE teacher_id = :teacher_id
  AND classroom_id = :classroom_id
  AND is_current = true;

-- 2. 旧バージョンを無効化
UPDATE teachers SET
  is_current = false,
  valid_to = NOW()
WHERE teacher_version_id = :old_version_id;

-- 3. 新バージョンを作成
INSERT INTO teachers (
  teacher_version_id, teacher_id, classroom_id, version_number,
  is_current, valid_from, change_reason_type, change_reason_note,
  name, gender, min_slots_per_week, max_slots_per_week,
  max_consecutive_slots, has_jhs_exam_experience, has_hs_exam_experience,
  university_rank, status
) VALUES (
  gen_random_uuid(), :teacher_id, :classroom_id, :old_version_number + 1,
  true, NOW(), :change_reason_type, :change_reason_note,
  :name, :gender, :min_slots_per_week, :max_slots_per_week,
  :max_consecutive_slots, :has_jhs_exam_experience, :has_hs_exam_experience,
  :university_rank, 'active'
) RETURNING teacher_version_id;

-- 4. teacher_subjectsを新バージョン用に再作成
INSERT INTO teacher_subjects (teacher_version_id, subject_id)
SELECT :new_version_id, unnest(:subject_ids::varchar[]);

-- 5. teacher_gradesを新バージョン用に再作成
INSERT INTO teacher_grades (teacher_version_id, grade)
SELECT :new_version_id, unnest(:grades::grade[]);

-- 6. NG関係を更新（差分のみ）
-- 既存を削除
DELETE FROM ng_relations WHERE teacher_id = :teacher_id;
-- 新規を追加
INSERT INTO ng_relations (ng_relation_id, teacher_id, student_id, created_by)
SELECT gen_random_uuid(), :teacher_id, unnest(:ng_student_ids::varchar[]), 'teacher';

-- 7. 監査ログ記録
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:user_id, 'UPDATE', 'Teacher', :teacher_id, :old_value_json, :new_value_json);

COMMIT;
```

### F043: 講師削除

論理削除（deleted_atフラグ）。確定済み時間割で使用中の場合は警告表示。

#### API仕様

```
DELETE /api/classrooms/{classroom_id}/teachers/{teacher_id}

成功レスポンス (200 OK):
{
  "message": "講師を削除しました"
}

警告レスポンス (200 OK with warning):
{
  "message": "講師を削除しました",
  "warning": "確定済み時間割で使用されています"
}
```

#### 事前チェック

```sql
-- 確定済み時間割での使用確認
SELECT COUNT(*) FROM schedule_slots ss
JOIN schedules s ON ss.schedule_id = s.schedule_id
WHERE ss.teacher_id = :teacher_id
  AND s.status = 'confirmed';
```

#### トランザクション処理

```sql
BEGIN;

-- 論理削除
UPDATE teachers SET
  deleted_at = NOW(),
  is_current = false,
  valid_to = NOW()
WHERE teacher_id = :teacher_id
  AND classroom_id = :classroom_id
  AND is_current = true;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value)
VALUES (:user_id, 'DELETE', 'Teacher', :teacher_id, :old_value_json);

COMMIT;
```

## 7. エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| VAL_010 | 氏名は必須です | 空値 |
| VAL_011 | 氏名は50文字以内で入力してください | 文字数超過 |
| VAL_012 | 指導可能科目を1つ以上選択してください | 未選択 |
| VAL_013 | 指導可能学年を1つ以上選択してください | 未選択 |
| VAL_014 | 最小コマ数は最大コマ数以下で設定してください | 不整合 |
| VAL_015 | 最大連続コマは1〜4の範囲で入力してください | 範囲外 |
| BIZ_010 | 指導可能科目と指導可能学年が整合していません | 学年カテゴリ不一致 |
