### F011: 充足率サマリー表示

#### 計算式

```
充足率 = (1対2コマ数 × 2) / (1対2コマ数 × 2 + 1対1コマ数) × 100
```

#### DBマッピング

**キャッシュ使用**:
- `schedules.fulfillment_rate` - 計算済みの値を直接参照

**リアルタイム計算（キャッシュ更新時）**:
```sql
SELECT
  COUNT(*) FILTER (WHERE slot_type = 'one_to_two') as one_to_two_count,
  COUNT(*) FILTER (WHERE slot_type = 'one_to_one') as one_to_one_count
FROM schedule_slots
WHERE schedule_id = :schedule_id;

-- 計算: (one_to_two_count * 2) / (one_to_two_count * 2 + one_to_one_count) * 100
```

#### API仕様

```
GET /api/classrooms/{classroom_id}/dashboard/fulfillment

レスポンス:
{
  "fulfillment_rate": 82.5,
  "one_to_two_slots": 40,
  "one_to_one_slots": 20,
  "total_student_slots": 100,
  "comparison": {
    "previous_week": 80.0,
    "change": 2.5,
    "direction": "up"
  }
}
```

#### エッジケース

| 状況 | 充足率 | 表示 |
|------|--------|------|
| 生徒が0人 | 計算対象外 | 「-」 |
| 全員1対2で配置 | 100% | 100% |
| 全員1対1で配置 | 0% | 0% |
| 未配置の生徒あり | 配置済みのみで計算 | N% |

### F012: 人員状況ヒートマップ表示

曜日×時限のマトリクスで需給状況を色分け表示。

#### 需給差の定義

```
需給差 = 供給枠数 - 需要人数
供給枠数 = シフト○の講師数 × 2
需要人数 = 受講希望○または△の生徒数
```

#### DBマッピング

**供給（講師）**:
```sql
SELECT day_of_week, slot_number, COUNT(*) as teacher_count
FROM teacher_shift_preferences
WHERE term_id = :term_id
  AND classroom_id = :classroom_id
  AND preference_value = 'available'
GROUP BY day_of_week, slot_number;
```

**需要（生徒）**:
```sql
SELECT day_of_week, slot_number, COUNT(*) as student_count
FROM student_preferences
WHERE term_id = :term_id
  AND classroom_id = :classroom_id
  AND preference_value IN ('preferred', 'possible')
GROUP BY day_of_week, slot_number;
```

#### API仕様

```
GET /api/classrooms/{classroom_id}/dashboard/heatmap

レスポンス:
{
  "cells": [
    {
      "day_of_week": "mon",
      "slot_number": 1,
      "supply": 6,
      "demand": 4,
      "balance": 2,
      "status": "sufficient"
    },
    ...
  ]
}
```

#### 表示色

| 需給差 | ステータス | 表示色 |
|--------|-----------|--------|
| >= 2 | sufficient | 緑 |
| 1 | normal | 黄緑 |
| 0 | tight | 黄 |
| < 0 | shortage | 赤 |

### F013: 科目別カバー率表示

#### 計算ロジック（按分方式）

```sql
-- ステップ1: 科目別需要を集計
SELECT subject_id, SUM(slots_per_week) as demand
FROM student_subjects ss
JOIN students s ON ss.student_version_id = s.student_version_id
WHERE s.classroom_id = :classroom_id
  AND s.is_current = true
  AND s.deleted_at IS NULL
GROUP BY subject_id;

-- ステップ2: 講師ごとの供給を按分計算
-- (講師の最大コマ数を、担当科目の需要比率で配分)
```

#### API仕様

```
GET /api/classrooms/{classroom_id}/dashboard/subject-coverage

レスポンス:
{
  "subjects": [
    {
      "subject_id": "JHS_MATH_PUB",
      "subject_name": "公立数学",
      "demand_slots": 30,
      "supply_slots": 25.5,
      "coverage_rate": 85.0,
      "status": "sufficient"
    },
    ...
  ]
}
```

#### 表示色

| カバー率 | ステータス | 表示色 |
|---------|-----------|--------|
| >= 100% | sufficient | 緑 |
| 70〜99% | partial | 黄 |
| < 70% | shortage | 赤 |

### F015: 通知・アラート表示

#### DBマッピング

**テーブル**: notifications

| カラム | 用途 |
|-------|------|
| user_id | 通知先ユーザー |
| classroom_id | 関連教室 |
| notification_type | 通知種別 |
| severity | 重要度 |
| is_read | 既読フラグ |

#### API仕様

```
GET /api/users/{user_id}/notifications

クエリパラメータ:
- classroom_id: 教室ID（フィルター）
- is_read: true/false（未読のみ表示時はfalse）
- limit: 件数（デフォルト10）

レスポンス:
{
  "notifications": [
    {
      "notification_id": "uuid",
      "notification_type": "fulfillment_low",
      "severity": "warning",
      "title": "充足率が目標を下回っています",
      "message": "4月期の充足率が75%です（目標85%）",
      "link_url": "/schedules/uuid",
      "is_read": false,
      "created_at": "2025-03-24T10:00:00Z"
    }
  ],
  "unread_count": 3
}
```

#### 既読更新API

```
PATCH /api/notifications/{notification_id}/read

レスポンス:
{
  "message": "既読にしました"
}

SQL:
UPDATE notifications SET is_read = true, read_at = NOW()
WHERE notification_id = :notification_id;
```

#### アラート条件

| 条件 | notification_type | severity |
|------|------------------|----------|
| 充足率が目標以下 | fulfillment_low | warning |
| 未回答の希望データあり | unanswered_preference | info |
| 確定期限が近い | term_deadline | critical |
| 時間割に競合あり | schedule_conflict | warning |
