## 4. 機能詳細

### F110: ユーザー一覧表示

#### 出力項目

| 項目名 | DBカラム | 説明 |
|--------|---------|------|
| 氏名 | users.name | ユーザーの氏名 |
| メールアドレス | users.email | ログイン用メールアドレス |
| 役割 | users.role | 教室長/エリアマネジャー/システム管理者 |
| 担当教室/エリア | user_classrooms | 担当範囲 |
| ステータス | users.is_active | 有効/無効 |
| 最終ログイン | users.last_login_at | 最後にログインした日時 |

#### API仕様

```
GET /api/users

クエリパラメータ:
- filter[role]: classroom_manager / area_manager / system_admin / all
- filter[is_active]: true / false / all
- search: 氏名・メールアドレスの部分一致
- page, per_page: ページネーション
- sort: user_name / -last_login_at

レスポンス:
{
  "data": [
    {
      "user_id": "uuid",
      "user_name": "山田 太郎",
      "email": "yamada@example.com",
      "role": "classroom_manager",
      "role_display": "教室長",
      "is_active": true,
      "assignments": [
        {"type": "classroom", "id": "uuid", "name": "渋谷教室"}
      ],
      "last_login_at": "2025-03-24T10:00:00Z"
    }
  ],
  "pagination": {...}
}
```

#### SQL例

```sql
SELECT
  u.user_id,
  u.user_name,
  u.email,
  u.role,
  u.is_active,
  u.last_login_at,
  COALESCE(
    json_agg(
      json_build_object(
        'type', CASE WHEN uc.area_id IS NOT NULL THEN 'area' ELSE 'classroom' END,
        'id', COALESCE(uc.area_id, c.classroom_id::text),
        'name', COALESCE(uc.area_id, c.classroom_name)
      )
    ) FILTER (WHERE uc.classroom_id IS NOT NULL),
    '[]'
  ) as assignments
FROM users u
LEFT JOIN user_classrooms uc ON u.user_id = uc.user_id AND uc.deleted_at IS NULL
LEFT JOIN classrooms c ON uc.classroom_id = c.classroom_id
WHERE u.deleted_at IS NULL
  AND (:role IS NULL OR u.role = :role)
  AND (:is_active IS NULL OR u.is_active = :is_active)
  AND (:search IS NULL OR u.name ILIKE '%' || :search || '%' OR u.email ILIKE '%' || :search || '%')
GROUP BY u.user_id
ORDER BY u.name
LIMIT :per_page OFFSET :offset;
```

### F111: ユーザー新規作成

#### 入力項目

| 項目名 | 型 | 必須 | DBカラム | バリデーション |
|--------|-----|------|---------|--------------|
| 氏名 | string | ○ | name | 最大100文字 |
| メールアドレス | string | ○ | email | メール形式、重複不可 |
| 役割 | enum | ○ | role | classroom_manager / area_manager / system_admin |
| 担当教室 | uuid[] | △ | user_classrooms | 教室長・エリアマネジャーの場合必須 |
| ステータス | boolean | ○ | is_active | デフォルト: true |

#### API仕様

```
POST /api/users

リクエスト:
{
  "name": "田中 次郎",
  "email": "tanaka@example.com",
  "role": "classroom_manager",
  "classroom_ids": ["uuid1", "uuid2"],
  "is_active": true
}

成功レスポンス (201 Created):
{
  "user_id": "uuid",
  "message": "ユーザーを作成しました。初期パスワードをメールで送信しました。"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 初期パスワード生成（アプリケーション側）
-- password = generate_random_password(12)
-- password_hash = bcrypt(password, cost=12)

-- ユーザー作成
INSERT INTO users (
  user_id, name, email, password_hash, role, is_active,
  force_password_change, login_failed_count
)
VALUES (
  gen_random_uuid(), :name, :email, :password_hash, :role, :is_active,
  true, 0
)
RETURNING user_id;

-- 担当教室を登録
INSERT INTO user_classrooms (user_classroom_id, user_id, classroom_id, area_id)
SELECT gen_random_uuid(), :user_id, classroom_id, NULL
FROM unnest(:classroom_ids::uuid[]) as classroom_id;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:admin_user_id, 'CREATE', 'User', :user_id, :new_value_json);

COMMIT;

-- トランザクション後: 初期パスワードをメール送信（非同期）
```

#### 初期パスワード仕様

- 生成ルール: 英大文字 + 英小文字 + 数字の混合、12文字
- 初回ログイン時に変更必須（force_password_change = true）

### F112: ユーザー情報編集

#### API仕様

```
PUT /api/users/{user_id}

リクエスト:
{
  "name": "田中 次郎（更新）",
  "email": "tanaka-new@example.com",
  "role": "classroom_manager",
  "classroom_ids": ["uuid1", "uuid3"],
  "is_active": true
}

成功レスポンス (200 OK):
{
  "user_id": "uuid",
  "message": "ユーザー情報を更新しました"
}

エラーレスポンス（自分自身の役割変更時、403 Forbidden）:
{
  "error": {
    "code": "BIZ_110",
    "message": "自分自身の役割は変更できません"
  }
}
```

#### トランザクション処理

```sql
BEGIN;

-- 自分自身の役割変更チェック（アプリケーション側）

-- 最後のシステム管理者チェック
SELECT COUNT(*) FROM users
WHERE role = 'system_admin' AND is_active = true AND deleted_at IS NULL;
-- 結果が1で、対象ユーザーがsystem_adminかつ役割変更しようとしている場合はエラー

-- ユーザー情報更新
UPDATE users SET
  name = :name,
  email = :email,
  role = :role,
  is_active = :is_active,
  updated_at = NOW()
WHERE user_id = :user_id;

-- 担当教室を更新（削除→再登録）
DELETE FROM user_classrooms WHERE user_id = :user_id;

INSERT INTO user_classrooms (user_classroom_id, user_id, classroom_id, area_id)
SELECT gen_random_uuid(), :user_id, classroom_id, NULL
FROM unnest(:classroom_ids::uuid[]) as classroom_id;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:admin_user_id, 'UPDATE', 'User', :user_id, :old_value_json, :new_value_json);

COMMIT;
```

#### 制約

| 制約 | エラーコード | 条件 |
|------|------------|------|
| 自分自身の役割変更不可 | BIZ_110 | 編集対象 = ログインユーザー かつ 役割を変更 |
| 最後の管理者は無効化/役割変更不可 | BIZ_111 | システム管理者が1人のみ かつ 無効化または役割変更 |

### F113: ユーザー削除

#### API仕様

```
DELETE /api/users/{user_id}

成功レスポンス (200 OK):
{
  "message": "ユーザーを削除しました"
}

削除不可時 (409 Conflict):
{
  "error": {
    "code": "BIZ_112",
    "message": "このユーザーは他のデータで参照されているため削除できません",
    "suggestion": "無効化をお勧めします"
  }
}

最後の管理者削除時 (409 Conflict):
{
  "error": {
    "code": "BIZ_113",
    "message": "最後のシステム管理者は削除できません"
  }
}
```

#### 処理分岐

```sql
-- 参照データ確認
SELECT EXISTS (
  SELECT 1 FROM audit_logs WHERE user_id = :user_id
  UNION
  SELECT 1 FROM schedules WHERE created_by = :user_id OR confirmed_by = :user_id
  UNION
  SELECT 1 FROM reschedules WHERE confirmed_by = :user_id
);

-- 参照なし: 論理削除
UPDATE users SET
  deleted_at = NOW(),
  email = email || '_deleted_' || NOW()::text  -- メール重複防止
WHERE user_id = :user_id;

-- 参照あり: 無効化を提案
-- エラーレスポンスを返却
```

### F114: パスワードリセット（管理者）

#### API仕様

```
POST /api/users/{user_id}/reset-password

成功レスポンス (200 OK):
{
  "message": "パスワードをリセットしました。新しいパスワードをメールで送信しました。"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 新パスワード生成（アプリケーション側）
-- new_password = generate_random_password(12)
-- new_password_hash = bcrypt(new_password, cost=12)

-- パスワード更新
UPDATE users SET
  password_hash = :new_password_hash,
  force_password_change = true,
  login_failed_count = 0,
  locked_until = NULL,
  updated_at = NOW()
WHERE user_id = :user_id;

-- 既存のリフレッシュトークンを無効化
UPDATE refresh_tokens SET
  revoked_at = NOW()
WHERE user_id = :user_id
  AND revoked_at IS NULL;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:admin_user_id, 'PASSWORD_RESET', 'User', :user_id, '{"by": "admin"}');

COMMIT;

-- トランザクション後: 新パスワードをメール送信（非同期）
```

## 7. エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| VAL_110 | 氏名は必須です | 空値 |
| VAL_111 | 氏名は100文字以内で入力してください | 文字数超過 |
| VAL_112 | メールアドレスの形式が正しくありません | 形式不正 |
| VAL_113 | このメールアドレスは既に使用されています | 重複 |
| VAL_114 | 担当教室を選択してください | 教室長/エリアマネジャーで未選択 |
| BIZ_110 | 自分自身の役割は変更できません | 自己役割変更 |
| BIZ_111 | 最後のシステム管理者の役割は変更できません | 最後の管理者役割変更 |
| BIZ_112 | このユーザーは他のデータで参照されているため削除できません | 参照あり削除 |
| BIZ_113 | 最後のシステム管理者は削除できません | 最後の管理者削除 |
