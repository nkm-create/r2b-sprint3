## 3. 機能詳細

### F001: ログイン

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|------|--------------|
| メールアドレス | string | ○ | RFC 5322準拠、最大255文字 |
| パスワード | string | ○ | 8〜72文字 |

#### DBマッピング

**参照テーブル**: users

| 入力項目 | DBカラム | 説明 |
|---------|---------|------|
| メールアドレス | users.email | UK制約あり |
| パスワード | users.password_hash | bcrypt(cost=12)でハッシュ |

**認証時の参照カラム**:

| カラム | チェック内容 |
|-------|-------------|
| status | 'active' であること |
| deleted_at | NULL であること |
| locked_until | NULL または 現在時刻より過去 |
| login_failed_count | 5未満 であること |

#### API仕様

```
POST /api/auth/login

リクエスト:
{
  "email": "user@example.com",
  "password": "********"
}

成功レスポンス (200 OK):
{
  "access_token": "eyJhbG...",
  "refresh_token": "eyJhbG...",
  "token_type": "Bearer",
  "expires_in": 3600,
  "user": {
    "user_id": "uuid",
    "email": "user@example.com",
    "name": "山田太郎",
    "role": "classroom_manager",
    "force_password_change": false
  }
}

エラーレスポンス (401 Unauthorized):
{
  "error": {
    "code": "AUTH_001",
    "message": "メールアドレスまたはパスワードが正しくありません"
  }
}
```

#### 処理フロー

```
1. 入力バリデーション
2. users.email で検索
3. ユーザー存在確認
   └─ 存在しない → AUTH_001エラー
4. ステータス確認
   └─ status != 'active' または deleted_at != NULL → AUTH_002エラー
5. アカウントロック確認
   └─ locked_until > NOW() → AUTH_003エラー（残り時間を返却）
6. パスワード照合（bcrypt.compare）
   └─ 不一致 → login_failed_count++、AUTH_001エラー
      └─ 5回目 → locked_until = NOW() + 30分
7. 認証成功
   └─ login_failed_count = 0
   └─ last_login_at = NOW()
8. JWT発行（access_token: 1時間、refresh_token: 7日）
9. refresh_tokens テーブルにリフレッシュトークン保存
10. audit_logs に LOGIN 記録
11. force_password_change 確認
    └─ true → 初回パスワード変更画面へリダイレクト
    └─ false → ダッシュボードへリダイレクト
```

#### トランザクション境界

```sql
BEGIN;
-- ログイン失敗時
UPDATE users SET
  login_failed_count = login_failed_count + 1,
  locked_until = CASE
    WHEN login_failed_count + 1 >= 5 THEN NOW() + INTERVAL '30 minutes'
    ELSE locked_until
  END
WHERE user_id = :user_id;

-- または ログイン成功時
UPDATE users SET
  login_failed_count = 0,
  last_login_at = NOW()
WHERE user_id = :user_id;

INSERT INTO refresh_tokens (user_id, token_hash, expires_at, device_info, ip_address)
VALUES (:user_id, :token_hash, NOW() + INTERVAL '7 days', :device_info, :ip_address);

INSERT INTO audit_logs (user_id, action, entity_type, ip_address, user_agent)
VALUES (:user_id, 'LOGIN', 'User', :ip_address, :user_agent);

COMMIT;
```

#### エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| AUTH_001 | メールアドレスまたはパスワードが正しくありません | 認証失敗 |
| AUTH_002 | このアカウントは無効です | status != 'active' |
| AUTH_003 | アカウントがロックされています。{N}分後に再試行してください | locked_until > NOW() |
| VAL_001 | メールアドレスの形式が正しくありません | 形式エラー |
| VAL_002 | パスワードは8文字以上で入力してください | 文字数不足 |

### F002: パスワードリマインダー

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|------|--------------|
| メールアドレス | string | ○ | RFC 5322準拠 |

#### DBマッピング

**参照テーブル**: users, password_reset_tokens

| 処理 | テーブル | カラム |
|------|---------|--------|
| ユーザー検索 | users | email |
| トークン発行 | password_reset_tokens | token_hash, expires_at |

#### API仕様

```
POST /api/auth/password-reset/request

リクエスト:
{
  "email": "user@example.com"
}

レスポンス (200 OK):
{
  "message": "パスワードリセットメールを送信しました"
}
```

**注意**: セキュリティ上、ユーザーが存在しない場合も同じレスポンスを返す

#### 処理フロー

```
1. 入力バリデーション
2. users.email で検索
3. ユーザーが存在する場合のみ:
   a. リセットトークン生成（SecureRandom, 32bytes）
   b. password_reset_tokens に保存（有効期限: 24時間）
   c. リセットメール送信（非同期）
4. 常に成功レスポンスを返す
```

#### リセット実行

```
POST /api/auth/password-reset/execute

リクエスト:
{
  "token": "reset_token_string",
  "new_password": "********"
}

成功レスポンス (200 OK):
{
  "message": "パスワードを変更しました"
}

エラーレスポンス (400 Bad Request):
{
  "error": {
    "code": "AUTH_004",
    "message": "トークンが無効または期限切れです"
  }
}
```

#### トランザクション境界

```sql
BEGIN;

-- トークン検証
SELECT user_id FROM password_reset_tokens
WHERE token_hash = :token_hash
  AND expires_at > NOW()
  AND used_at IS NULL;

-- パスワード更新
UPDATE users SET
  password_hash = :new_password_hash,
  force_password_change = false,
  login_failed_count = 0,
  locked_until = NULL
WHERE user_id = :user_id;

-- トークン使用済みにマーク
UPDATE password_reset_tokens SET used_at = NOW()
WHERE token_hash = :token_hash;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id)
VALUES (:user_id, 'UPDATE', 'User', :user_id);

COMMIT;
```

### F003: ログイン状態維持

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|------|--------------|
| リフレッシュトークン | string | ○ | 有効期限内、未失効 |

#### DBマッピング

**参照テーブル**: refresh_tokens

| 処理 | DBカラム | 説明 |
|------|---------|------|
| トークン検証 | refresh_tokens.token_hash | ハッシュ比較 |
| 有効期限確認 | refresh_tokens.expires_at | 現在時刻と比較 |
| 失効確認 | refresh_tokens.revoked_at | NULLであること |

#### API仕様

```
POST /api/auth/refresh

リクエスト:
{
  "refresh_token": "eyJhbG..."
}

成功レスポンス (200 OK):
{
  "access_token": "eyJhbG...",
  "token_type": "Bearer",
  "expires_in": 3600
}

エラーレスポンス (401 Unauthorized):
{
  "error": {
    "code": "AUTH_006",
    "message": "セッションが無効です。再ログインしてください"
  }
}
```

#### 処理フロー

```
1. リフレッシュトークンのハッシュ計算
2. refresh_tokens テーブルで検索
   └─ 存在しない → AUTH_006エラー
3. 有効期限確認
   └─ expires_at < NOW() → AUTH_006エラー
4. 失効確認
   └─ revoked_at != NULL → AUTH_006エラー
5. ユーザーステータス確認
   └─ status != 'active' → AUTH_006エラー
6. 新アクセストークン発行（1時間有効）
7. レスポンス返却
```

#### エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| AUTH_006 | セッションが無効です。再ログインしてください | トークン無効・期限切れ・失効済み |

### F004: 初回パスワード変更

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション |
|--------|-----|------|--------------|
| 現在のパスワード | string | ○ | 現在のパスワードと一致 |
| 新パスワード | string | ○ | 8〜72文字、英数字混合 |
| 新パスワード（確認） | string | ○ | 新パスワードと一致 |

#### DBマッピング

**更新テーブル**: users

| 処理 | DBカラム |
|------|---------|
| パスワード更新 | password_hash |
| フラグ解除 | force_password_change = false |

#### API仕様

```
POST /api/auth/password/change

ヘッダー:
Authorization: Bearer {access_token}

リクエスト:
{
  "current_password": "********",
  "new_password": "********"
}

成功レスポンス (200 OK):
{
  "message": "パスワードを変更しました"
}
```

#### エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| AUTH_005 | 現在のパスワードが正しくありません | 照合失敗 |
| VAL_003 | 新パスワードは8文字以上で入力してください | 文字数不足 |
| VAL_004 | 新パスワードは英字と数字を含めてください | 形式エラー |
| VAL_005 | 確認用パスワードが一致しません | 不一致 |

## 4. UI状態遷移

```
[ログイン画面]
    ├─→ ログイン成功
    │     ├─→ force_password_change=true → [初回パスワード変更]
    │     │                                    └─→ 変更完了 → [ダッシュボード]
    │     └─→ force_password_change=false → [ダッシュボード]
    │
    ├─→ ログイン失敗 → [エラーメッセージ表示]
    │
    └─→ パスワードを忘れた → [パスワードリマインダー画面]
                              ├─→ メール送信完了 → [確認メッセージ]
                              └─→ メールのリンククリック → [パスワード再設定画面]
                                                           └─→ 設定完了 → [ログイン画面]
```

## 5. セキュリティ要件

| 項目 | 仕様 |
|------|------|
| パスワードハッシュ | bcrypt (cost=12) |
| アカウントロック | 5回失敗で30分ロック |
| リセットトークン有効期限 | 24時間 |
| アクセストークン有効期限 | 1時間 |
| リフレッシュトークン有効期限 | 7日 |
| HTTPS | 必須 |
| CSRF対策 | トークン方式 |

---

## 6. 非機能要件（画面固有）

| 項目 | 目標値 |
|------|--------|
| ログイン処理時間 | 1秒以内 |
| パスワードハッシュ計算時間 | 200ms以内 |
| 同時ログインユーザー数 | 100ユーザー |
