## 3. DBマッピング

### 3.1 関連テーブル

| テーブル | 役割 |
|---------|------|
| students | 生徒マスタ（バージョン管理） |
| student_subjects | 生徒-科目別コマ数（中間） |
| ng_relations | NG関係 |
| subjects | 科目マスタ（参照のみ） |
| teachers | 講師マスタ（希望講師参照） |

### 3.2 カラムマッピング

**students テーブル**:

| 画面項目 | DBカラム | 型 | 制約 |
|---------|---------|-----|------|
| 氏名 | name | VARCHAR(50) | NN |
| 学年 | grade | grade | NN, ele1〜hs3 |
| 最大連続コマ | max_consecutive_slots | INTEGER | NN, 1〜4 |
| 希望講師 | preferred_teacher_id | VARCHAR(20) | FK |
| 講師希望性別 | preferred_teacher_gender | preferred_gender | male/female/any |
| 志望レベル | aspiration_level | aspiration_level | A/B/C |
| 通塾目的 | enrollment_purpose | enrollment_purpose | hs_exam/jhs_exam/internal/remedial/other |
| ステータス | status | student_status | active/inactive |

**student_subjects テーブル**:

| 画面項目 | DBカラム | 型 | 制約 |
|---------|---------|-----|------|
| 科目 | subject_id | VARCHAR(20) | FK |
| コマ数/週 | slots_per_week | INTEGER | NN, 1以上 |

| カラム | 用途 |
|-------|------|
| student_version_id | バージョンごとのPK（UUID） |

## 4. 機能詳細

### F050: 生徒一覧表示

#### 出力項目

| 項目名 | DBカラム | 説明 |
|--------|---------|------|
| 氏名 | students.name | 生徒の氏名 |
| 学年 | students.grade | 小1〜高3 |
| 受講科目 | student_subjects → subjects.subject_name | 科目一覧（省略表示） |
| 総コマ数 | SUM(student_subjects.slots_per_week) | 週あたりの総受講コマ数 |
| 志望レベル | students.aspiration_level | A/B/C |
| ステータス | students.status | active→有効 / inactive→無効 |

#### API仕様

```
GET /api/classrooms/{classroom_id}/students

クエリパラメータ:
- page: ページ番号
- per_page: 件数
- q: 検索文字列（氏名部分一致）
- filter[status]: active / inactive / all
- filter[grade]: ele1〜hs3 / elementary / junior_high / high_school
- filter[subject_id]: 科目IDで絞り込み
- filter[aspiration_level]: A / B / C
- filter[enrollment_purpose]: hs_exam / jhs_exam / internal / remedial / other
- sort: name / -created_at など

レスポンス:
{
  "data": [
    {
      "student_id": "S001",
      "student_version_id": "uuid",
      "version_number": 2,
      "name": "鈴木一郎",
      "grade": "jhs2",
      "subjects": [
        {"subject_id": "JHS_MATH_PUB", "subject_name": "公立数学", "slots_per_week": 2},
        {"subject_id": "JHS_ENG", "subject_name": "中学英語", "slots_per_week": 2}
      ],
      "total_slots_per_week": 4,
      "max_consecutive_slots": 2,
      "preferred_teacher_id": "T001",
      "preferred_teacher_name": "山田太郎",
      "preferred_teacher_gender": "any",
      "ng_teacher_ids": ["T005"],
      "aspiration_level": "A",
      "enrollment_purpose": "hs_exam",
      "status": "active"
    }
  ],
  "pagination": {...}
}
```

#### SQL例

```sql
SELECT
  s.student_id,
  s.student_version_id,
  s.version_number,
  s.name,
  s.grade,
  s.max_consecutive_slots,
  s.preferred_teacher_id,
  t.name as preferred_teacher_name,
  s.preferred_teacher_gender,
  s.aspiration_level,
  s.enrollment_purpose,
  s.status,
  COALESCE(SUM(ss.slots_per_week), 0) as total_slots_per_week,
  array_agg(DISTINCT ss.subject_id) as subject_ids
FROM students s
LEFT JOIN student_subjects ss ON s.student_version_id = ss.student_version_id
LEFT JOIN teachers t ON s.preferred_teacher_id = t.teacher_id
  AND t.is_current = true AND t.deleted_at IS NULL
WHERE s.classroom_id = :classroom_id
  AND s.is_current = true
  AND s.deleted_at IS NULL
GROUP BY s.student_version_id, t.name
ORDER BY s.created_at DESC
LIMIT :per_page OFFSET :offset;
```

### F051: 生徒新規登録

#### 入力項目

| 項目名 | 型 | 必須 | DBカラム | バリデーション |
|--------|-----|------|---------|--------------|
| 氏名 | string | ○ | name | 最大50文字 |
| 学年 | enum | ○ | grade | ele1〜hs3 |
| 科目別コマ数 | object[] | ○ | student_subjects | [{subject_id, slots_per_week}]、1件以上 |
| 最大連続コマ | number | ○ | max_consecutive_slots | 1〜4 |
| 希望講師 | string | - | preferred_teacher_id | 講師マスタから選択 |
| NG講師 | string[] | - | ng_relations | 講師マスタから選択 |
| 講師希望性別 | enum | - | preferred_teacher_gender | male / female / any |
| 志望レベル | enum | - | aspiration_level | A / B / C |
| 通塾目的 | enum | - | enrollment_purpose | hs_exam / jhs_exam / internal / remedial / other |

#### API仕様

```
POST /api/classrooms/{classroom_id}/students

リクエスト:
{
  "name": "鈴木一郎",
  "grade": "jhs2",
  "subjects": [
    {"subject_id": "JHS_MATH_PUB", "slots_per_week": 2},
    {"subject_id": "JHS_ENG", "slots_per_week": 2}
  ],
  "max_consecutive_slots": 2,
  "preferred_teacher_id": "T001",
  "ng_teacher_ids": ["T005"],
  "preferred_teacher_gender": "any",
  "aspiration_level": "A",
  "enrollment_purpose": "hs_exam"
}

成功レスポンス (201 Created):
{
  "student_id": "S001",
  "student_version_id": "uuid",
  "version_number": 1,
  "message": "生徒を登録しました"
}
```

#### バリデーション: 学年と科目の整合性

```sql
-- 学年カテゴリを取得
SELECT grade_category FROM subjects WHERE subject_id IN (:subject_ids);

-- 生徒の学年カテゴリと一致することを確認
-- ele1〜ele6 → elementary
-- jhs1〜jhs3 → junior_high
-- hs1〜hs3 → high_school
```

#### トランザクション処理

```sql
BEGIN;

-- 1. student_id採番
SELECT COALESCE(MAX(CAST(SUBSTRING(student_id FROM 2) AS INTEGER)), 0) + 1 AS next_num
FROM students WHERE classroom_id = :classroom_id;

-- 2. studentsに新規バージョン作成
INSERT INTO students (
  student_version_id, student_id, classroom_id, version_number,
  is_current, valid_from, change_reason_type,
  name, grade, max_consecutive_slots,
  preferred_teacher_id, preferred_teacher_gender,
  aspiration_level, enrollment_purpose, status
) VALUES (
  gen_random_uuid(), :student_id, :classroom_id, 1,
  true, NOW(), 'INITIAL',
  :name, :grade, :max_consecutive_slots,
  :preferred_teacher_id, :preferred_teacher_gender,
  :aspiration_level, :enrollment_purpose, 'active'
) RETURNING student_version_id;

-- 3. student_subjectsに科目別コマ数を登録
INSERT INTO student_subjects (student_version_id, subject_id, slots_per_week)
VALUES (:student_version_id, :subject_id, :slots_per_week);
-- 複数科目分を繰り返し

-- 4. NG関係を登録
INSERT INTO ng_relations (ng_relation_id, teacher_id, student_id, created_by)
SELECT gen_random_uuid(), unnest(:ng_teacher_ids::varchar[]), :student_id, 'student';

-- 5. 監査ログ記録
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Student', :student_id, :new_value_json);

COMMIT;
```

### F052: 生徒情報編集

#### トランザクション処理

```sql
BEGIN;

-- 1. 現在のバージョンを取得
SELECT student_version_id, version_number
FROM students
WHERE student_id = :student_id
  AND classroom_id = :classroom_id
  AND is_current = true;

-- 2. 旧バージョンを無効化
UPDATE students SET
  is_current = false,
  valid_to = NOW()
WHERE student_version_id = :old_version_id;

-- 3. 新バージョンを作成
INSERT INTO students (
  student_version_id, student_id, classroom_id, version_number,
  is_current, valid_from, change_reason_type, change_reason_note,
  name, grade, max_consecutive_slots,
  preferred_teacher_id, preferred_teacher_gender,
  aspiration_level, enrollment_purpose, status
) VALUES (...) RETURNING student_version_id;

-- 4. student_subjectsを新バージョン用に再作成
INSERT INTO student_subjects (student_version_id, subject_id, slots_per_week)
VALUES (:new_version_id, :subject_id, :slots_per_week);

-- 5. NG関係を更新
DELETE FROM ng_relations WHERE student_id = :student_id;
INSERT INTO ng_relations (ng_relation_id, teacher_id, student_id, created_by)
SELECT gen_random_uuid(), unnest(:ng_teacher_ids::varchar[]), :student_id, 'student';

-- 6. 監査ログ記録
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:user_id, 'UPDATE', 'Student', :student_id, :old_value_json, :new_value_json);

COMMIT;
```

### F053: 生徒削除

#### API仕様

```
DELETE /api/classrooms/{classroom_id}/students/{student_id}

事前チェック:
- 確定済み時間割に含まれる場合 → 警告
- 振替待ちデータがある場合 → 警告
```

#### 事前チェックSQL

```sql
-- 確定済み時間割での使用確認
SELECT COUNT(*) FROM schedule_slots ss
JOIN schedules s ON ss.schedule_id = s.schedule_id
WHERE (ss.student1_id = :student_id OR ss.student2_id = :student_id)
  AND s.status = 'confirmed';

-- 振替待ち確認
SELECT COUNT(*) FROM absences a
WHERE a.absent_person_id = :student_id
  AND a.absent_type = 'student'
  AND a.substitution_status = 'pending';
```

## 8. エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| VAL_020 | 氏名は必須です | 空値 |
| VAL_021 | 氏名は50文字以内で入力してください | 文字数超過 |
| VAL_022 | 受講科目を1つ以上登録してください | 未登録 |
| VAL_023 | コマ数は1以上で入力してください | 範囲外 |
| VAL_024 | 最大連続コマは1〜4の範囲で入力してください | 範囲外 |
| VAL_025 | 希望講師とNG講師に同じ講師が設定されています | 重複 |
| BIZ_020 | 学年と受講科目が整合していません | 学年カテゴリ不一致 |
| BIZ_021 | 指定された講師は有効ではありません | 無効な講師参照 |
