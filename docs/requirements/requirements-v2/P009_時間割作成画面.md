# P009: 時間割作成画面 要件定義書 v2

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | P009 |
| 画面名 | 時間割作成画面 |
| 対象ユーザー | 教室長 |
| 目的 | AIソルバーによる一発最適化で、人間では到達困難な高品質の時間割を短時間で生成 |

---

## 2. 設計思想：AI能力の最大活用

### 2.1 基本方針

| 方針 | 説明 |
|------|------|
| **一発最適解** | CP-SATソルバーが制約を満たしながら目的関数を最大化し、数十秒で最適解を生成 |
| **人間の介入最小化** | 基本は「生成→確認→確定」の3ステップ。例外時のみ調整 |
| **説明可能性** | なぜその解になったか、LLMが自然言語で説明 |
| **スーパーバイズ** | Orchestratorがソルバーを監視し、進捗・戦略切り替えを制御 |

### 2.2 価値提案

> 「教室長は、ターム選択→生成ボタンを押すだけで、数十秒で最適な時間割が生成される。人間が何時間もかけても到達できない品質の解を、AIが一瞬で出す。あとは確認して確定するだけ。」

### 2.3 従来手法との比較

| 観点 | 従来（人手/段階的改善） | 本システム（一発最適解） |
|------|----------------------|----------------------|
| 所要時間 | 数時間〜数日 | 数十秒〜1分 |
| 解の品質 | 担当者の経験に依存 | 数学的に最適化 |
| 再現性 | 低い | 高い（同一入力→同一結果） |
| 人的負荷 | 高い | 低い |

## 5. エージェントアーキテクチャ

### 5.1 設計原則

CP-SATソルバーの最適化能力を最大限に活かし、LLMは「説明」と「例外処理」に特化する。
Orchestratorがソルバーを監視・制御し、最適な戦略で求解を実行する。

```
┌─────────────────────────────────────────────────────────────────┐
│                    Schedule Orchestrator                        │
│                      【状態マシン】                              │
│                                                                 │
│   ・ソルバー監視（コールバック経由）                               │
│   ・早期終了判定（目標達成/停滞/タイムアウト）                       │
│   ・戦略切り替え判定（standard → relaxed → partial）              │
│   ・進捗のリアルタイム通知（WebSocket）                            │
└─────────────────────────────────────────────────────────────────┘
        │              │              │
        ▼              ▼              ▼
┌───────────┐  ┌───────────────┐  ┌───────────┐
│ Analyzer  │  │    Solver     │  │ Explainer │
│【ルール】  │  │ 【CP-SAT】    │  │ 【LLM】   │
│           │  │               │  │           │
│ 問題分析   │  │ 求解実行      │  │ 結果説明   │
│ 難易度判定 │  │ コールバック   │  │ 緩和提案   │
│ 戦略提案   │  │ 進捗報告      │  │ What-if   │
└───────────┘  └───────────────┘  └───────────┘
```

### 5.2 各コンポーネントの技術方式

| コンポーネント | 実装方式 | LLM活用 | 主な処理 |
|--------------|---------|---------|---------|
| **Orchestrator** | 状態マシン | ❌ 不使用 | ソルバー監視、戦略切り替え、進捗通知 |
| **Analyzer** | ルールベース | ❌ 不使用 | 問題規模分析、ボトルネック検出、初期戦略提案 |
| **Solver** | OR-Tools CP-SAT | ❌ 不使用 | 制約充足問題として定式化し求解 |
| **Explainer** | LLM | ✅ 全面使用 | 結果説明、緩和提案、What-if分析 |

### 5.3 Schedule Orchestrator（全体統括）

**役割**: Solverを監視し、進捗報告・早期終了・戦略切り替えを制御
**実装方式**: 状態マシン + CP-SATコールバック

#### 責務一覧

| 責務 | 説明 | 実装方式 |
|------|------|---------|
| ソルバー監視 | コールバックで解発見を検知 | CpSolverSolutionCallback |
| 早期終了判定 | 目標達成/停滞/タイムアウト判定 | ルール |
| 戦略切り替え | standard→relaxed→partial | 状態マシン |
| 進捗通知 | 現在の充足率・経過時間を通知 | WebSocket |
| エラーリカバリー | 失敗時のハンドリング・部分解保存 | ルール |

#### 早期終了判定ロジック

```
解発見時（コールバック）:
├─ 充足率 ≥ 目標値（85%）？
│   └─ YES → 探索停止（目標達成）
├─ 前回解から10秒以上経過 & 改善幅 < 1%？
│   └─ YES → 探索停止（改善停滞）
├─ タイムアウト間近（残り5秒）？
│   └─ YES → 探索停止（タイムアウト）
└─ 上記以外 → 探索継続
```

#### 戦略切り替えロジック

```
┌─────────────┐     30秒経過      ┌─────────────┐    20秒経過     ┌─────────────┐
│  standard   │───解なし───────▶│  relaxed    │──解なし──────▶│  partial    │
│ （標準求解） │                  │（ソフト緩和）│                │（部分求解）  │
└─────────────┘                  └─────────────┘                └─────────────┘
      │                                │                              │
      │ 解発見                          │ 解発見                        │ 解発見
      ▼                                ▼                              ▼
   [成功]                           [成功]                          [部分成功]
   status: optimal                 status: feasible                status: partial
```

#### 戦略の定義

| 戦略 | タイムアウト | 特徴 |
|------|------------|------|
| standard | 30秒 | 全制約を適用、最適解を探索 |
| relaxed | 20秒 | ソフト制約の重みを50%に低減、実行可能解を優先 |
| partial | 20秒 | 難配置生徒を除外、部分解を構築 |

#### トランザクション処理

```sql
BEGIN;

-- 新しい時間割バージョンを作成
INSERT INTO schedules (
  schedule_id, term_id, classroom_id, version, status,
  fulfillment_rate, soft_constraint_rate, created_by
)
SELECT
  gen_random_uuid(), :term_id, :classroom_id,
  COALESCE(MAX(version), 0) + 1, 'draft',
  :fulfillment_rate, :soft_constraint_rate, :user_id
FROM schedules
WHERE term_id = :term_id
RETURNING schedule_id, version;

-- 生成ログを記録
INSERT INTO schedule_generation_logs (
  log_id, schedule_id, strategy, solution_status,
  fulfillment_rate, soft_constraint_rate,
  processing_time_ms, solutions_found, optimality_gap,
  orchestrator_decisions
)
VALUES (
  gen_random_uuid(), :schedule_id, :strategy, :solution_status,
  :fulfillment_rate, :soft_constraint_rate,
  :processing_time_ms, :solutions_found, :optimality_gap,
  :orchestrator_decisions_json
);

-- コマを一括挿入
INSERT INTO schedule_slots (
  slot_id, schedule_id, day_of_week, slot_number,
  teacher_id, student1_id, student1_subject_id,
  student2_id, student2_subject_id, slot_type, slot_status
)
VALUES (...);

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Schedule', :schedule_id, :new_value_json);

COMMIT;
```

## 7. LLM実装詳細

### 7.1 Explainer Agent

**役割**: 結果の説明、緩和提案、What-if分析

**プロンプト設計（結果説明）**:
```
システムプロンプト:
あなたは学習塾の時間割最適化結果を分析し、
教室長にわかりやすく説明するアシスタントです。

最適化の結果を以下の観点で説明してください：
1. 全体の評価（充足率、ソフト制約達成率）
2. 主なボトルネックとその構造的原因
3. トレードオフの選択肢（もしあれば）

ユーザープロンプト:
最適化結果:
- 充足率: {fulfillment_rate}%
- ソフト制約達成率: {soft_constraint_rate}%
- ボトルネック: {bottlenecks}
- 使用戦略: {strategy}
- 終了理由: {termination_reason}

この結果を教室長向けに説明してください。

出力形式: JSON
{
  "summary": {
    "overall": "...",
    "key_points": ["..."]
  },
  "bottleneck_explanation": {
    "main_bottleneck": "...",
    "detail": "...",
    "structural_cause": true/false
  },
  "trade_offs": [...]
}
```

**プロンプト設計（緩和提案）**:
```
システムプロンプト:
あなたは学習塾の時間割作成で実行不可能な状況を分析し、
最小限の影響で解決する緩和案を提案するアシスタントです。

制約の種類:
- H001-H005: ハード制約（緩和には慎重な判断が必要）
- S001-S007: ソフト制約（比較的緩和しやすい）

ユーザープロンプト:
実行不可能な理由:
{infeasibility_analysis}

関連データ:
{context}

影響が最小限の緩和案を優先度順に3つ提案してください。

出力形式: JSON
{
  "suggestions": [
    {
      "title": "...",
      "constraint_to_relax": "...",
      "impact": "...",
      "affected_count": N,
      "priority": "high/medium/low"
    }
  ]
}
```

### 7.2 フォールバック機構

| 状況 | 対応 |
|------|------|
| LLM呼び出し失敗 | テンプレートベース出力にフォールバック |
| レスポンス遅延（>3秒） | タイムアウト → テンプレート出力 |
| 出力パースエラー | 再試行（最大2回）→ テンプレート出力 |

**フォールバックテンプレート例**:

| 状況 | テンプレート |
|------|-------------|
| 最適解達成 | 「充足率{rate}%の最適解が見つかりました」 |
| 部分解 | 「{N}名が未配置です。手動調整が必要です」 |
| 実行不可能 | 「現在の制約では解が存在しません。制約の緩和が必要です」 |
| ボトルネック | 「{時間帯}に{科目}を教えられる講師が不足しています」 |
