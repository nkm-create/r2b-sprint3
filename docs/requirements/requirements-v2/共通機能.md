## 2. 検索・フィルター（全一覧画面共通）

### 2.1 テキスト検索

| 項目 | 仕様 |
|------|------|
| 検索対象 | 名前での部分一致 |
| 検索タイミング | 入力確定時（Enter）またはデバウンス（300ms） |
| 最小文字数 | 1文字以上 |
| 大文字/小文字 | 区別しない |

#### API仕様

```
GET /api/{resource}?q={検索文字列}

クエリパラメータ:
- q: 検索文字列（部分一致）

SQL実装:
WHERE LOWER(name) LIKE LOWER('%' || :q || '%')
```

### 2.2 フィルター

| 項目 | 仕様 |
|------|------|
| 複数条件 | AND条件で絞り込み |
| フィルタークリア | 「リセット」ボタンで全解除 |
| フィルター保存 | ブラウザのローカルストレージに保存（任意） |

#### API仕様

```
GET /api/{resource}?filter[status]=active&filter[gender]=male

SQL実装:
WHERE status = :status AND gender = :gender
```

### 2.3 ソート

| 項目 | 仕様 |
|------|------|
| 対象 | 各列での昇順/降順 |
| デフォルト | 作成日時の降順 |
| 複数列ソート | 非対応 |

#### API仕様

```
GET /api/{resource}?sort=-created_at

パラメータ:
- sort: カラム名（-プレフィックスで降順）

例:
- sort=name → ORDER BY name ASC
- sort=-created_at → ORDER BY created_at DESC
```

## 3. ページネーション

### 3.1 仕様

| 項目 | 仕様 |
|------|------|
| 1ページあたり件数 | 20件（デフォルト） |
| 選択可能件数 | 10 / 20 / 50 / 100件 |
| 表示形式 | 「1-20 / 92件」形式 |
| ページ移動 | 前へ/次へ、ページ番号直接指定 |

### 3.2 API仕様

```
GET /api/{resource}?page=1&per_page=20

リクエスト:
- page: ページ番号（1始まり）
- per_page: 1ページあたり件数

レスポンス:
{
  "data": [...],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total_count": 92,
    "total_pages": 5
  }
}

SQL実装:
LIMIT :per_page OFFSET (:page - 1) * :per_page
```

### 3.3 URL同期

- ページ番号、件数をURLクエリパラメータに反映
- ブックマーク・共有が可能

## 4. 削除処理

### 4.1 論理削除

| 項目 | 仕様 |
|------|------|
| 方式 | deleted_atフラグによる論理削除 |
| 削除日時 | deleted_atにタイムスタンプを記録 |
| 一覧表示 | 削除済みデータは非表示 |

#### DBマッピング

| 処理 | 対象カラム | 値 |
|------|----------|-----|
| 削除実行 | deleted_at | NOW() |
| 一覧取得 | WHERE条件 | deleted_at IS NULL |

### 4.2 復元

| 項目 | 仕様 |
|------|------|
| 復元権限 | システム管理者のみ |
| 復元操作 | deleted_atをnullに設定 |
| 復元履歴 | 監査ログに記録 |

#### API仕様

```
POST /api/{resource}/{id}/restore

レスポンス:
- 200 OK: 復元成功
- 403 Forbidden: 権限なし

SQL実装:
UPDATE {table} SET deleted_at = NULL WHERE id = :id
```

## 5. CSVエクスポート

### 5.1 対応画面

| 対象画面 | エクスポート内容 | ファイル名形式 |
|---------|----------------|---------------|
| P005 講師管理 | 講師一覧（全件またはフィルター適用後） | teachers_YYYYMMDD.csv |
| P006 生徒管理 | 生徒一覧（全件またはフィルター適用後） | students_YYYYMMDD.csv |
| P009 時間割作成 | 時間割データ（全体/講師別/生徒別） | schedule_{term_id}_YYYYMMDD.csv |

### 5.2 API仕様

```
GET /api/{resource}/export?format=csv&scope=all

クエリパラメータ:
- format: csv（固定）
- scope: all（全件）/ filtered（フィルター適用後）
- filter[*]: フィルター条件（scope=filteredの場合）

レスポンス:
- Content-Type: text/csv; charset=utf-8
- Content-Disposition: attachment; filename="teachers_20260324.csv"
```

### 5.3 CSV形式

| 項目 | 仕様 |
|------|------|
| 文字コード | UTF-8（BOM付き、Excel対応） |
| 区切り文字 | カンマ |
| ヘッダー行 | あり |
| 日付形式 | YYYY-MM-DD |
| 改行 | CRLF |
| 引用符 | ダブルクォート（値にカンマ・改行を含む場合） |

## 6. データバージョン管理

### 6.1 バージョン管理が必要なデータ

| データ | 対象テーブル | 変更頻度 | バージョン管理 |
|--------|------------|---------|--------------|
| 講師マスタ | teachers | 中 | **必須** |
| 生徒マスタ | students | 中 | **必須** |
| 時間割 | schedules | 高 | **必須** |
| 希望データ | teacher_shift_preferences, student_preferences | 中 | 推奨 |
| 制約条件 | term_constraints, policies | 低 | 推奨 |

### 6.2 マスタのバージョン管理

#### DBマッピング（講師マスタの場合）

| カラム | 型 | 用途 |
|-------|-----|------|
| teacher_version_id | UUID | PK（バージョンごと） |
| teacher_id | VARCHAR(20) | 不変ID（例: T001） |
| version_number | INTEGER | バージョン番号（1〜） |
| is_current | BOOLEAN | 現在有効なバージョンか |
| valid_from | TIMESTAMP | 有効開始日時 |
| valid_to | TIMESTAMP | 有効終了日時 |
| change_reason_type | ENUM | 変更理由種別 |
| change_reason_note | VARCHAR(500) | 変更理由補足 |

#### 変更理由種別（change_reason_type）

**講師マスタの場合（teacher_change_reason）**:

| 値 | 説明 |
|-----|------|
| SCHEDULE_CHANGE | シフト・コマ数の変更 |
| SUBJECT_CHANGE | 指導可能科目の変更 |
| GRADE_CHANGE | 指導可能学年の変更 |
| NG_CHANGE | NG生徒の追加・削除 |
| STATUS_CHANGE | ステータス変更（有効/無効） |
| INITIAL | 初回登録 |
| OTHER | その他 |

**生徒マスタの場合（student_change_reason）**:

| 値 | 説明 |
|-----|------|
| COURSE_CHANGE | 受講科目・コマ数の変更 |
| GRADE_UP | 学年進級 |
| PREFERENCE_CHANGE | 希望講師・NG講師の変更 |
| GOAL_CHANGE | 志望レベル・通塾目的の変更 |
| STATUS_CHANGE | ステータス変更（有効/無効） |
| INITIAL | 初回登録 |
| OTHER | その他 |

※ `OTHER` を選択した場合は `change_reason_note` の入力を必須とする。

#### トランザクション処理

```sql
-- バージョン更新時のトランザクション
BEGIN;

-- 1. 旧バージョンを無効化
UPDATE teachers
SET is_current = false, valid_to = NOW()
WHERE teacher_id = :teacher_id AND is_current = true;

-- 2. 新バージョンを作成
INSERT INTO teachers (
  teacher_version_id, teacher_id, classroom_id, version_number,
  is_current, valid_from, change_reason_type, ...
) VALUES (
  gen_random_uuid(), :teacher_id, :classroom_id, :new_version,
  true, NOW(), :change_reason_type, ...
);

-- 3. 監査ログ記録
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:user_id, 'UPDATE', 'Teacher', :teacher_id, :old_value, :new_value);

COMMIT;
```

### 6.3 時間割バージョン管理

#### DBマッピング

| カラム | 型 | 用途 |
|-------|-----|------|
| schedule_id | UUID | PK |
| term_id | UUID | タームID |
| version | INTEGER | バージョン番号 |
| status | schedule_status | draft / confirmed / archived |
| parent_version_id | UUID | 派生元バージョンID |
| master_snapshot | JSONB | 作成時のマスタバージョン参照 |
| generation_config | JSONB | 生成設定 |

#### ステータス遷移

```
draft → confirmed → archived
         ↓
       派生（新バージョン作成）
```

| ステータス | 説明 | 可能な操作 |
|-----------|------|----------|
| draft | 作成中 | 編集、削除、確定 |
| confirmed | 確定済み | 参照、PDF出力、派生（新バージョン作成） |
| archived | 過去ターム | 参照のみ |

## 8. バリデーションルール

### 8.1 学年と科目の整合性

| 学年カテゴリ | 設定可能な科目ID |
|------------|-----------------|
| 小学生（ele1〜ele6） | ELE_* |
| 中学生（jhs1〜jhs3） | JHS_* |
| 高校生（hs1〜hs3） | HS_* |

### 8.2 NG関係の相互性

**テーブル**: ng_relations

| カラム | 型 | 説明 |
|-------|-----|------|
| ng_relation_id | UUID | PK |
| teacher_id | VARCHAR(20) | 講師ID |
| student_id | VARCHAR(20) | 生徒ID |
| created_by | ng_created_by | teacher / student |
| created_at | TIMESTAMP | 作成日時 |

**処理**:
- 講師側でNG生徒を設定 → ng_relationsに1レコード追加
- 生徒側でNG講師を設定 → 同一レコードで管理（created_byで判別）
- 削除時は該当レコードを削除

### 8.3 コマ数の整合性

**DB CHECK制約**:
```sql
-- teachers: コマ数の整合性
ALTER TABLE teachers ADD CONSTRAINT chk_teachers_slots
  CHECK (min_slots_per_week <= max_slots_per_week);

ALTER TABLE teachers ADD CONSTRAINT chk_teachers_consecutive
  CHECK (max_consecutive_slots BETWEEN 1 AND 4);

-- students: 連続コマ数の範囲
ALTER TABLE students ADD CONSTRAINT chk_students_consecutive
  CHECK (max_consecutive_slots BETWEEN 1 AND 4);

-- student_subjects: コマ数は1以上
ALTER TABLE student_subjects ADD CONSTRAINT chk_student_subjects_slots
  CHECK (slots_per_week >= 1);
```

## 9. エラーハンドリング

### 9.1 エラー分類

| 分類 | HTTPステータス | 説明 |
|------|--------------|------|
| バリデーションエラー | 400 Bad Request | 入力値の形式・範囲エラー |
| 認証エラー | 401 Unauthorized | 未認証 |
| 認可エラー | 403 Forbidden | 権限不足 |
| リソースなし | 404 Not Found | 対象データが存在しない |
| 競合エラー | 409 Conflict | 重複登録、楽観的ロック失敗 |
| システムエラー | 500 Internal Server Error | サーバー・通信エラー |

### 9.2 エラーコード体系

| プレフィックス | 対象 | 例 |
|--------------|------|-----|
| AUTH_ | 認証・認可 | AUTH_001（ログイン失敗） |
| VAL_ | バリデーション | VAL_001（必須項目未入力） |
| BIZ_ | ビジネスルール | BIZ_001（重複登録） |
| SYS_ | システム | SYS_001（タイムアウト） |

### 9.3 エラーレスポンス形式

```json
{
  "error": {
    "code": "VAL_001",
    "message": "必須項目が入力されていません",
    "details": [
      {
        "field": "name",
        "message": "氏名は必須です"
      }
    ]
  }
}
```

### 9.4 エラー表示

| エラー種別 | 表示方法 |
|-----------|---------|
| フィールドエラー | 該当フィールドの下に赤字表示 |
| フォームエラー | フォーム上部にサマリー表示 |
| システムエラー | トースト通知 |

## 11. 監査ログ

### 11.1 DBマッピング

**テーブル**: audit_logs

| カラム | 型 | 説明 |
|-------|-----|------|
| log_id | UUID | PK |
| user_id | UUID | 操作ユーザーID |
| action | VARCHAR(50) | CREATE / UPDATE / DELETE / LOGIN |
| entity_type | VARCHAR(50) | エンティティ種別 |

| old_value | JSONB | 変更前値 |
| new_value | JSONB | 変更後値 |

| user_agent | VARCHAR(500) | ユーザーエージェント |
| created_at | TIMESTAMP | 作成日時 |

### 11.2 記録対象操作

| 操作 | action値 | old_value | new_value |
|------|---------|-----------|-----------|
| 新規作成 | CREATE | null | 作成データ |
| 更新 | UPDATE | 更新前データ | 更新後データ |
| 削除 | DELETE | 削除データ | null |
| ログイン | LOGIN | null | {ip, user_agent} |
