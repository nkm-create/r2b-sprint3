## 4. 機能詳細

### F100: ターム一覧表示

#### 出力項目

| 項目名 | DBカラム/計算 | 説明 |
|--------|-------------|------|
| ターム名 | terms.term_name | タームの名称（例: 2025年4月期） |
| 期間 | terms.start_date〜terms.end_date | 開始日〜終了日 |
| ステータス | terms.status | "作成中" / "確定" / "アーカイブ" |
| 充足率 | schedules.fulfillment_rate | confirmed時間割の充足率 |
| 時間割バージョン数 | COUNT(schedules) | 作成されたバージョン数 |

#### API仕様

```
GET /api/classrooms/{classroom_id}/terms

クエリパラメータ:
- page, per_page: ページネーション
- filter[status]: creating / confirmed / archived / all
- sort: -start_date（デフォルト）

レスポンス:
{
  "data": [
    {
      "term_id": "uuid",
      "term_name": "2025年4月期",
      "start_date": "2025-04-01",
      "end_date": "2025-04-30",
      "status": "confirmed",
      "fulfillment_rate": 85.5,
      "schedule_version_count": 3,
      "has_overlap": false
    }
  ],
  "pagination": {...}
}
```

#### SQL例

```sql
SELECT
  t.term_id,
  t.term_name,
  t.start_date,
  t.end_date,
  t.status,
  s.fulfillment_rate,
  (SELECT COUNT(*) FROM schedules WHERE term_id = t.term_id) as schedule_version_count,
  EXISTS (
    SELECT 1 FROM terms t2
    WHERE t2.classroom_id = t.classroom_id
      AND t2.term_id != t.term_id
      AND t2.deleted_at IS NULL
      AND (t2.start_date, t2.end_date) OVERLAPS (t.start_date, t.end_date)
  ) as has_overlap
FROM terms t
LEFT JOIN schedules s ON t.term_id = s.term_id AND s.status = 'confirmed'
WHERE t.classroom_id = :classroom_id
  AND t.deleted_at IS NULL
ORDER BY t.start_date DESC
LIMIT :per_page OFFSET :offset;
```

#### ステータス説明

| ステータス | 説明 | DB値 |
|-----------|------|------|
| 作成中 | 時間割を作成・調整中 | creating |
| 確定 | 時間割が確定済み | confirmed |
| アーカイブ | 過去のターム（参照のみ） | archived |

### F102: ターム複製

元タームの条件設定をコピーして新ターム作成。

#### API仕様

```
POST /api/classrooms/{classroom_id}/terms/{term_id}/duplicate

リクエスト:
{
  "new_term_name": "2025年5月期",
  "start_date": "2025-05-01",
  "end_date": "2025-05-31",
  "copy_options": {
    "term_constraints": true,
    "policies": true,
    "teacher_snapshot": false,
    "student_snapshot": false,
    "preferences": false
  }
}

成功レスポンス (201 Created):
{
  "term_id": "new-uuid",
  "term_name": "2025年5月期",
  "message": "タームを複製しました"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 1. 新タームを作成
INSERT INTO terms (
  term_id, classroom_id, term_name, start_date, end_date, status
) VALUES (
  gen_random_uuid(), :classroom_id, :new_term_name, :start_date, :end_date, 'creating'
) RETURNING term_id;

-- 2. 制約条件をコピー（copy_options.term_constraints = true の場合）
INSERT INTO term_constraints (
  constraint_id, term_id, target_type, target_id, constraint_type, constraint_value
)
SELECT
  gen_random_uuid(), :new_term_id, target_type, target_id, constraint_type, constraint_value
FROM term_constraints
WHERE term_id = :source_term_id;

-- 3. ポリシーをコピー（copy_options.policies = true の場合）
INSERT INTO policies (
  policy_id, term_id, policy_type, is_enabled, parameters
)
SELECT
  gen_random_uuid(), :new_term_id, policy_type, is_enabled, parameters
FROM policies
WHERE term_id = :source_term_id;

-- 4. 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Term', :new_term_id, :new_value_json);

COMMIT;
```

#### コピー対象選択

| 項目 | デフォルト | 説明 |
|------|----------|------|
| 制約条件 | ○ | term_constraintsのコピー |
| ポリシー | ○ | policiesのコピー |
| 講師マスタ | × | 講師データのスナップショット |
| 生徒マスタ | × | 生徒データのスナップショット |
| 希望データ | × | シフト・受講希望のコピー |

### F103: ターム編集

#### 入力項目

| 項目名 | 型 | 必須 | DBカラム | バリデーション |
|--------|-----|------|---------|--------------|
| ターム名 | string | ○ | term_name | 最大50文字、同一教室内で重複不可 |
| 開始日 | date | ○ | start_date | 終了日より前 |
| 終了日 | date | ○ | end_date | 開始日より後 |

#### API仕様

```
PUT /api/classrooms/{classroom_id}/terms/{term_id}

リクエスト:
{
  "term_name": "2025年4月期（修正）",
  "start_date": "2025-04-01",
  "end_date": "2025-04-30"
}

成功レスポンス (200 OK):
{
  "term_id": "uuid",
  "message": "タームを更新しました"
}

警告レスポンス（確定済み時間割がある場合）:
{
  "term_id": "uuid",
  "message": "タームを更新しました",
  "warning": "確定済み時間割があるため、期間変更の影響を確認してください"
}
```

#### 制約

- 確定済み時間割がある場合、期間の変更は警告表示

#### 期間重複のルール

**同一教室内でのターム期間の重複は許可する。**

| 状況 | 許可 | 理由 |
|------|-----|------|
| 期間が完全に同じ | ○ | 複数パターンの時間割を比較検討する用途 |
| 期間が部分的に重複 | ○ | 移行期間で両方のタームを参照する用途 |
| 同一ターム名 | × | 混乱を避けるためターム名は一意とする |

**重複時の表示**:
- ターム一覧画面で期間が重複するタームには「重複あり」のラベルを表示
- 重複するターム同士をグループ表示（同じ背景色でハイライト）

**運用上の注意**:
- 「確定」ステータスにできるのは、同一期間内で1タームのみ
- 他のタームが確定済みの期間に新タームを確定しようとした場合、警告を表示

### F104: ターム削除

論理削除。確定済み時間割がある場合はアーカイブ化。

#### API仕様

```
DELETE /api/classrooms/{classroom_id}/terms/{term_id}

処理分岐:
- 時間割なし → 論理削除
- draft時間割のみ → 確認後削除
- confirmed時間割あり → 削除不可、アーカイブ化を提案

レスポンス（削除可能時）:
{
  "message": "タームを削除しました"
}

レスポンス（削除不可時、409 Conflict）:
{
  "error": {
    "code": "BIZ_100",
    "message": "確定済み時間割があるため削除できません",
    "suggestion": "アーカイブ化をお勧めします"
  }
}
```

#### 処理

| 条件 | 動作 |
|------|------|
| 時間割なし | 完全削除（論理削除） |
| draft時間割のみ | 確認後削除 |
| confirmed時間割あり | アーカイブ化（削除不可） |

#### アーカイブ化API

```
POST /api/classrooms/{classroom_id}/terms/{term_id}/archive

レスポンス:
{
  "term_id": "uuid",
  "message": "タームをアーカイブしました"
}
```

## 7. エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| VAL_100 | ターム名は必須です | 空値 |
| VAL_101 | ターム名は50文字以内で入力してください | 文字数超過 |
| VAL_102 | 開始日は終了日より前に設定してください | 日付不整合 |
| BIZ_100 | 確定済み時間割があるため削除できません | 削除不可 |
| BIZ_101 | 同一教室内でターム名が重複しています | 名前重複 |
| BIZ_102 | この期間に既に確定済みタームが存在します | 期間重複で確定不可 |
