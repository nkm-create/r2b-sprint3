# 非機能要件定義書 v2

## 1. 概要

本ドキュメントは、学習塾時間割最適化システムの非機能要件を定義する。

### 1.1 v1からの追加内容

| 追加項目 | 説明 |
|---------|------|
| DBとの関連 | 各要件を実現するDB設計との対応 |
| 監視・アラート | 運用監視の詳細仕様 |
| LLM非機能要件 | Advisor/Evaluatorエージェントの要件 |

---

## 2. パフォーマンス要件

### 2.1 レスポンス時間

| 画面/機能 | 目標値 | 対応するDB設計 |
|----------|--------|--------------|
| ログイン | 1秒以内 | idx_users_email インデックス |
| ダッシュボード表示 | 1秒以内 | schedules.fulfillment_rate キャッシュ |
| 一覧画面（20件） | 500ms以内 | 部分インデックス (is_current = true) |
| 登録・更新処理 | 1秒以内 | - |
| 時間割生成（一発求解） | 60秒以内 | Orchestrator戦略切り替え含む |

### 2.2 DB設計での対応

| 要件 | DB設計 |
|------|--------|
| 現行バージョンの高速取得 | `idx_teachers_current`, `idx_students_current` (WHERE is_current = true) |
| ターム別データの高速取得 | `idx_teacher_prefs_term`, `idx_student_prefs_term` |
| 充足率の即時表示 | schedules.fulfillment_rate, soft_constraint_rate (キャッシュカラム) |
| 未読通知の高速取得 | `idx_notifications_user_unread` (WHERE is_read = false) |

### 2.3 同時接続数

| 項目 | 目標値 | 制約 |
|------|--------|------|
| 同時ログインユーザー | 100ユーザー | RDS db.t4g.micro の最大接続数 |
| 同時時間割生成 | 1教室につき1セッション | 排他制御による |

---

## 3. 可用性要件

### 3.1 稼働率

| 項目 | 目標値 |
|------|--------|
| 月間稼働率 | 99.5%（月間約3.6時間のダウンタイム許容） |
| 計画メンテナンス | 月1回、深夜2時〜4時 |

### 3.2 データ保護

| 項目 | 目標値 | 実装 |
|------|--------|------|
| RPO（目標復旧時点） | 1時間 | RDS自動バックアップ（5分間隔） |
| RTO（目標復旧時間） | 4時間 | スナップショットからの復元 |
| バックアップ保持期間 | 7日間 | RDS設定 |

### 3.3 障害対応

| 障害種別 | 対応 |
|---------|------|
| DBフェイルオーバー | Single-AZ（自動復旧なし）、手動対応 |
| アプリケーション障害 | ECS自動再起動 |
| LLM API障害 | テンプレートベースにフォールバック |

---

## 4. セキュリティ要件

### 4.1 認証・認可

| 項目 | 仕様 | DBカラム |
|------|------|---------|
| パスワードハッシュ | bcrypt (cost=12) | users.password_hash |
| アカウントロック | 5回失敗で30分ロック | users.login_failed_count, locked_until |
| パスワードリセット有効期限 | 24時間 | password_reset_tokens.expires_at |
| アクセストークン有効期限 | 1時間 | JWT exp claim |
| リフレッシュトークン有効期限 | 7日 | refresh_tokens.expires_at |
| 初回パスワード変更強制 | 有効 | users.force_password_change |

### 4.2 データ保護

| 項目 | 仕様 |
|------|------|
| 通信暗号化 | TLS 1.2以上（HTTPS必須） |
| 機密データ暗号化 | OAuthトークン（AES-256-GCM） |
| SQLインジェクション対策 | パラメータ化クエリ |
| XSS対策 | 出力エスケープ、CSP |
| CSRF対策 | トークン方式 |

### 4.3 監査ログ

| 項目 | 仕様 | DBテーブル |
|------|------|----------|
| 記録対象 | 全CRUD操作、ログイン | audit_logs |
| 保持期間 | 1年 | パーティショニング推奨 |
| 記録内容 | ユーザーID、操作、変更前後値、IP、UA | audit_logs.* |

---

## 5. データ整合性要件

### 5.1 トランザクション管理

| 処理 | トランザクション境界 |
|------|-------------------|
| マスタ更新 | バージョン無効化 + 新バージョン作成 + 監査ログ |
| 時間割確定 | ステータス更新 + 確定者記録 |
| ターム複製 | ターム作成 + 制約コピー + ポリシーコピー |

### 5.2 バージョン管理

| 対象 | 管理方式 | 関連テーブル |
|------|---------|------------|
| 講師マスタ | version_number + is_current | teachers, teacher_subjects, teacher_grades |
| 生徒マスタ | version_number + is_current | students, student_subjects |
| 時間割 | version + status | schedules, schedule_slots |

### 5.3 論理削除

| 対象テーブル | 削除カラム |
|------------|----------|
| users | deleted_at |
| classrooms | deleted_at |
| teachers | deleted_at |
| students | deleted_at |
| terms | deleted_at |

## 6. LLM非機能要件

### 6.1 対象エージェント

| エージェント | LLM用途 |
|------------|---------|
| Explainer | 結果説明、緩和提案、What-if分析、ボトルネック説明 |

### 6.2 パフォーマンス要件

| 項目 | 目標値 |
|------|--------|
| レスポンス時間 | 500ms〜1.5秒 |
| タイムアウト | 3秒 |
| リトライ回数 | 最大2回 |

### 6.3 可用性要件

| 項目 | 目標値 |
|------|--------|
| 可用性 | 99.5%（フォールバック込み） |
| フォールバック | テンプレートベース出力 |

### 6.4 コスト要件

| 項目 | 上限 |
|------|------|
| 月額コスト | $500以下（1000教室想定） |
| 1リクエストあたり | 〜$0.50 |

### 6.5 推奨プロバイダ

| プロバイダ | モデル | 理由 |
|----------|--------|------|
| Anthropic | Claude 3 | 説明生成に優れる、日本語対応 |

### 6.6 フォールバック条件

| 条件 | 対応 |
|------|------|
| LLM API障害 | テンプレートベース出力 |
| レスポンス遅延（>3秒） | タイムアウト → テンプレート出力 |
| 出力パースエラー | リトライ（最大2回）→ テンプレート出力 |
