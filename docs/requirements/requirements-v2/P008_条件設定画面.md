# P008: 条件設定画面 要件定義書 v2

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | P008 |
| 画面名 | 条件設定画面 |
| 対象ユーザー | 教室長 |
| 目的 | マスタデータから自動反映された制約を確認し、ターム固有の調整と全体ポリシーを設定 |

---

## 2. 設計思想：マスタデータとの役割分担

| データソース | 管理場所 | P008での扱い |
|-------------|---------|--------------|
| 講師の基本情報 | P005 講師管理 | **自動反映**（参照のみ） |
| 生徒の基本情報 | P006 生徒管理 | **自動反映**（参照のみ） |
| 教室設定 | P004 教室設定 | **自動反映**（参照のみ） |
| シフト希望・受講希望 | P007 希望データ | **自動反映**（参照のみ） |
| **ターム固有の追加調整** | P008 条件設定 | **編集可能** |
| **全体ポリシー** | P008 条件設定 | **編集可能** |

### 2.1 一発求解方式における条件設定の重要性

P009（時間割作成画面）では**一発求解方式**を採用しており、AIソルバーが設定された制約・ポリシーに基づいて最適解を自動生成する。
このため、**生成前の条件設定が最終結果を大きく左右する**。

| 観点 | 説明 |
|------|------|
| 事前設定の重要性 | 生成後の手動調整は例外処理。基本は条件設定で意図を反映 |
| ポリシーの影響 | P001〜P006の有効/無効、パラメータがソルバーの目的関数に直接反映 |
| ターム固有調整 | 特定の講師・生徒に対する制約調整で、ボトルネックを事前に解消 |

> **推奨**: 新タームの条件設定時は、前タームの生成結果（ボトルネック分析）を参考に調整を行う。

### 4.1 関連テーブル

### 4.2 term_constraints カラムマッピング

| カラム | 型 | 説明 |
|-------|-----|------|
| constraint_id | UUID | PK |
| term_id | UUID | FK→terms |
| target_type | constraint_target_type | teacher / student / classroom |
| target_id | VARCHAR(50) | 対象ID |
| constraint_type | constraint_type | 制約種別 |
| constraint_value | JSONB | 制約値 |

**constraint_type の値と constraint_value 構造**:

| constraint_type | 対象 | constraint_value例 |
|-----------------|------|-------------------|
| max_slots | teacher/student | {"value": 5} |
| min_slots | teacher | {"value": 2} |
| max_consecutive | teacher/student | {"value": 3} |
| subject_limit | teacher | {"subject_ids": ["JHS_MATH_PUB"]} |
| day_limit | teacher | {"days": ["mon", "wed", "fri"]} |
| preferred_teacher | student | {"teacher_id": "T001"} |
| ng_teacher | student | {"teacher_ids": ["T002", "T003"]} |
| gender_preference | student | {"gender": "male"} |
| booth_capacity | classroom | {"value": 6} |

### 4.3 policies カラムマッピング

## 6. 機能詳細

### F070: ターム選択・作成

#### API仕様

```
POST /api/classrooms/{classroom_id}/terms

リクエスト:
{
  "term_name": "2025年4月期",
  "start_date": "2025-04-01",
  "end_date": "2025-04-30"
}

成功レスポンス (201 Created):
{
  "term_id": "uuid",
  "term_name": "2025年4月期",
  "message": "タームを作成しました"
}
```

#### トランザクション処理

```sql
BEGIN;

-- ターム作成
INSERT INTO terms (term_id, classroom_id, term_name, start_date, end_date, status)
VALUES (gen_random_uuid(), :classroom_id, :term_name, :start_date, :end_date, 'creating')
RETURNING term_id;

-- デフォルトポリシーを設定
INSERT INTO policies (policy_id, term_id, policy_type, is_enabled, parameters)
VALUES
  (gen_random_uuid(), :term_id, 'P001', true, :p001_default),
  (gen_random_uuid(), :term_id, 'P002', true, :p002_default),
  ...;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Term', :term_id, :new_value_json);

COMMIT;
```

### F072: ターム固有調整の設定

#### API仕様

**講師調整の保存**:
```
POST /api/classrooms/{classroom_id}/terms/{term_id}/constraints

リクエスト:
{
  "target_type": "teacher",
  "target_id": "T001",
  "constraints": [
    {"constraint_type": "max_slots", "value": {"value": 3}},
    {"constraint_type": "subject_limit", "value": {"subject_ids": ["ELE_MATH_JKN"]}}
  ]
}

成功レスポンス (201 Created):
{
  "message": "制約を保存しました",
  "constraint_ids": ["uuid1", "uuid2"]
}
```

#### トランザクション処理

```sql
BEGIN;

-- 既存の同一対象の制約を削除
DELETE FROM term_constraints
WHERE term_id = :term_id
  AND target_type = :target_type
  AND target_id = :target_id;

-- 新しい制約を登録
INSERT INTO term_constraints (constraint_id, term_id, target_type, target_id, constraint_type, constraint_value)
VALUES (gen_random_uuid(), :term_id, :target_type, :target_id, :constraint_type, :constraint_value);
-- 複数制約分を繰り返し

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:user_id, 'UPDATE', 'TermConstraint', :term_id, :old_value, :new_value);

COMMIT;
```

### F073: 全体ポリシーの設定

#### API仕様

```
PUT /api/classrooms/{classroom_id}/terms/{term_id}/policies

リクエスト:
{
  "policies": [
    {
      "policy_type": "P001",
      "is_enabled": true,
      "parameters": {
        "level_A": ["A"],
        "level_B": ["A", "B"],
        "level_C": ["A", "B", "C"]
      }
    },
    {
      "policy_type": "P005",
      "is_enabled": true,
      "parameters": {
        "target_rate": 85
      }
    }
  ]
}

成功レスポンス (200 OK):
{
  "message": "ポリシーを保存しました"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 既存ポリシーを更新
UPDATE policies SET
  is_enabled = :is_enabled,
  parameters = :parameters,
  updated_at = NOW()
WHERE term_id = :term_id
  AND policy_type = :policy_type;

-- 存在しない場合は作成（UPSERT）
INSERT INTO policies (policy_id, term_id, policy_type, is_enabled, parameters)
VALUES (gen_random_uuid(), :term_id, :policy_type, :is_enabled, :parameters)
ON CONFLICT (term_id, policy_type) DO UPDATE SET
  is_enabled = :is_enabled,
  parameters = :parameters,
  updated_at = NOW();

COMMIT;
```

### F074: 条件の保存・読み込み

#### API仕様

**テンプレート保存**:
```
POST /api/classrooms/{classroom_id}/policy-templates

リクエスト:
{
  "template_name": "標準設定",

```


```
POST /api/classrooms/{classroom_id}/terms/{term_id}/copy-from

リクエスト:
{
  "source_term_id": "uuid",
  "copy_constraints": true,
  "copy_policies": true
}
```

```
[条件設定画面]
    │
    ├─→ ターム選択ドロップダウン
```

| VAL_072 | コマ数上限はマスタ設定値以下で設定してください | マスタ値超過 |
