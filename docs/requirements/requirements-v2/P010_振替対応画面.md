# P010: 振替対応画面 要件定義書 v2

## 1. 画面概要

| 項目 | 内容 |
|------|------|
| 画面ID | P010 |
| 画面名 | 振替対応画面 |
| 対象ユーザー | 教室長 |
| 目的 | 講師・生徒の欠席登録と振替の手配 |

## 4. 機能詳細

### F090: 欠席登録

#### 入力項目

| 項目名 | 型 | 必須 | DBカラム | バリデーション |
|--------|-----|------|---------|--------------|
| 対象者種別 | enum | ○ | target_type | teacher / student |
| 対象者 | string | ○ | target_id | マスタに存在する講師/生徒ID |
| 欠席コマ | uuid | ○ | original_slot_id | 確定済み時間割のコマID |
| 欠席理由 | string | - | reason | 最大200文字 |
| 振替希望有無 | boolean | ○ | reschedule_requested | デフォルト: true |

#### API仕様

**対象者のコマ一覧取得**:
```
GET /api/classrooms/{classroom_id}/schedules/{schedule_id}/slots

クエリパラメータ:
- target_type: teacher / student
- target_id: 対象者ID

レスポンス:
{
  "slots": [
    {
      "slot_id": "uuid",
      "day_of_week": "mon",
      "slot_number": 1,
      "subject_name": "中受算数",
      "teacher_name": "田村 悠樹",
      "students": ["山田 太郎", "高橋 花子"]
    }
  ]
}
```

**欠席登録**:
```
POST /api/classrooms/{classroom_id}/absences

リクエスト:
{
  "target_type": "student",
  "target_id": "S001",
  "original_slot_id": "uuid",
  "reason": "体調不良",
  "reschedule_requested": true
}

成功レスポンス (201 Created):
{
  "absence_id": "uuid",
  "message": "欠席を登録しました",
  "next_action": "reschedule_candidates"
}
```

#### トランザクション処理

```sql
BEGIN;

-- 欠席登録
INSERT INTO absences (
  absence_id, classroom_id, schedule_id, target_type, target_id,
  original_slot_id, reason, reschedule_requested, status
)
VALUES (
  gen_random_uuid(), :classroom_id, :schedule_id, :target_type, :target_id,
  :original_slot_id, :reason, :reschedule_requested, 'pending'
)
RETURNING absence_id;

-- 元コマのステータスを更新
UPDATE schedule_slots SET
  slot_status = 'absent',
  updated_at = NOW()
WHERE slot_id = :original_slot_id;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Absence', :absence_id, :new_value_json);

COMMIT;
```

### F091: 振替候補提案

#### API仕様

```
GET /api/classrooms/{classroom_id}/absences/{absence_id}/reschedule-candidates

クエリパラメータ:
- limit: 候補数（デフォルト: 10）

レスポンス:
{
  "absence": {
    "absence_id": "uuid",
    "target_type": "student",
    "target_name": "山田 太郎",
    "original_slot": {
      "day_of_week": "mon",
      "slot_number": 1,
      "teacher_name": "田村 悠樹",
      "subject_name": "中受算数"
    }
  },
  "candidates": [
    {
      "candidate_id": 1,
      "slot": {
        "day_of_week": "wed",
        "slot_number": 2,
        "date": "2025-03-26"
      },
      "reschedule_type": "time_change",
      "teacher": {
        "teacher_id": "T001",
        "teacher_name": "田村 悠樹",
        "is_original": true
      },
      "priority_score": 90,
      "score_breakdown": [
        {"reason": "元講師維持", "score": 50},
        {"reason": "生徒希望○", "score": 30},
        {"reason": "同週内", "score": 10}
      ],
      "slot_type": "one_to_two",
      "partner_student": "高橋 花子"
    },
    {
      "candidate_id": 2,
      "slot": {...},
      "reschedule_type": "substitute",
      "teacher": {
        "teacher_id": "T002",
        "teacher_name": "佐藤 次郎",
        "is_original": false
      },
      "priority_score": 65,
      ...
    }
  ]
}
```

#### 候補検索ロジック

**欠席パターン別処理**:

| パターン | 検索条件 |
|---------|----------|
| 生徒欠席 | 同じ講師の空きコマ + 生徒の希望時間帯 |
| 講師欠席 | 別講師（同科目対応）の空きコマ + 影響生徒の希望時間帯 |

**候補検索SQL（生徒欠席の場合）**:
```sql
-- ステップ1: 元講師の空きスロットを検索
SELECT
  ss.slot_id,
  ss.day_of_week,
  ss.slot_number,
  'time_change' as reschedule_type,
  true as is_original_teacher,
  sp.preference_value as student_preference
FROM schedule_slots ss
JOIN student_preferences sp
  ON sp.day_of_week = ss.day_of_week
  AND sp.slot_number = ss.slot_number
  AND sp.student_id = :student_id
  AND sp.term_id = :term_id
WHERE ss.schedule_id = :schedule_id
  AND ss.teacher_id = :original_teacher_id
  AND ss.slot_status = 'active'
  AND (ss.slot_type = 'empty' OR (ss.slot_type = 'one_to_one' AND can_add_student(ss.slot_id)))
  AND sp.preference_value IN ('preferred', 'possible');

-- ステップ2: 代講可能な講師の空きスロットを検索
SELECT
  ss.slot_id,
  ss.day_of_week,
  ss.slot_number,
  'substitute' as reschedule_type,
  false as is_original_teacher,
  sp.preference_value as student_preference
FROM schedule_slots ss
JOIN teachers t ON ss.teacher_id = t.teacher_id
JOIN teacher_subjects tsub ON t.teacher_version_id = tsub.teacher_version_id
JOIN student_preferences sp
  ON sp.day_of_week = ss.day_of_week
  AND sp.slot_number = ss.slot_number
  AND sp.student_id = :student_id
WHERE ss.schedule_id = :schedule_id
  AND tsub.subject_id = :subject_id
  AND ss.slot_status = 'active'
  AND ss.slot_type IN ('empty', 'one_to_one')
  AND sp.preference_value IN ('preferred', 'possible');
```

#### 優先度スコア計算

| 要素 | スコア | 条件 |
|------|--------|------|
| 元講師維持 | +50 | is_original_teacher = true |
| 生徒希望○ | +30 | student_preference = 'preferred' |
| 生徒希望△ | +10 | student_preference = 'possible' |
| 日程近さ（同週内） | +20 | date <= original_date + 7 |
| 日程近さ（翌週） | +10 | date <= original_date + 14 |
| 1対2が可能 | +15 | slot_type = 'one_to_one' |

### F092: 振替確定

#### API仕様

```
POST /api/classrooms/{classroom_id}/absences/{absence_id}/reschedule

リクエスト:
{
  "candidate_id": 1,
  "slot_id": "uuid",
  "notify_parties": true
}

成功レスポンス (201 Created):
{
  "reschedule_id": "uuid",
  "message": "振替を確定しました",
  "details": {
    "original_slot": "月曜1限",
    "new_slot": "水曜2限",
    "teacher": "田村 悠樹",
    "type": "time_change"
  }
}
```

#### トランザクション処理

```sql
BEGIN;

-- 振替記録を作成
INSERT INTO reschedules (
  reschedule_id, absence_id, new_slot_id, reschedule_type,
  confirmed_by, confirmed_at
)
VALUES (
  gen_random_uuid(), :absence_id, :new_slot_id, :reschedule_type,
  :user_id, NOW()
)
RETURNING reschedule_id;

-- 欠席ステータスを更新
UPDATE absences SET
  status = 'rescheduled',
  updated_at = NOW()
WHERE absence_id = :absence_id;

-- 振替先コマに生徒を追加（または新規コマ作成）
-- パターン1: 既存の1対1に追加 → 1対2に変更
UPDATE schedule_slots SET
  slot_type = 'one_to_two',
  student2_id = :student_id,
  updated_at = NOW()
WHERE slot_id = :new_slot_id
  AND slot_type = 'one_to_one';

-- パターン2: 空きコマに新規作成
INSERT INTO schedule_slots (...)
SELECT ...
WHERE NOT EXISTS (SELECT 1 FROM schedule_slots WHERE slot_id = :new_slot_id);

-- 充足率を再計算してキャッシュ更新
UPDATE schedules SET
  fulfillment_rate = (
    SELECT (COUNT(*) FILTER (WHERE slot_type = 'one_to_two') * 2.0) /
           NULLIF(COUNT(*) FILTER (WHERE slot_type = 'one_to_two') * 2 +
                  COUNT(*) FILTER (WHERE slot_type = 'one_to_one'), 0) * 100
    FROM schedule_slots
    WHERE schedule_id = :schedule_id
  ),
  updated_at = NOW()
WHERE schedule_id = :schedule_id;

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, new_value)
VALUES (:user_id, 'CREATE', 'Reschedule', :reschedule_id, :new_value_json);

COMMIT;
```

### F093: 欠席・振替履歴表示

#### API仕様

```
GET /api/classrooms/{classroom_id}/absences

クエリパラメータ:
- filter[target_type]: teacher / student / all
- filter[status]: pending / rescheduled / cancelled / all
- filter[start_date]: 期間開始日
- filter[end_date]: 期間終了日
- page, per_page: ページネーション
- sort: -created_at（デフォルト）

レスポンス:
{
  "data": [
    {
      "absence_id": "uuid",
      "target_type": "student",
      "target_name": "山田 太郎",
      "original_slot": {
        "day_of_week": "mon",
        "slot_number": 1,
        "date": "2025-03-24",
        "teacher_name": "田村 悠樹",
        "subject_name": "中受算数"
      },
      "reason": "体調不良",
      "status": "rescheduled",
      "reschedule": {
        "new_slot": {
          "day_of_week": "wed",
          "slot_number": 2,
          "date": "2025-03-26"
        },
        "type": "time_change",
        "confirmed_at": "2025-03-24T11:00:00Z"
      },
      "created_at": "2025-03-24T10:00:00Z"
    }
  ],
  "summary": {
    "total": 15,
    "pending": 2,
    "rescheduled": 12,
    "cancelled": 1
  },
  "pagination": {...}
}
```

#### SQL例

```sql
SELECT
  a.absence_id,
  a.target_type,
  CASE
    WHEN a.target_type = 'teacher' THEN t.teacher_name
    WHEN a.target_type = 'student' THEN s.student_name
  END as target_name,
  ss.day_of_week as original_day,
  ss.slot_number as original_slot,
  a.reason,
  a.status,
  r.reschedule_id,
  r.reschedule_type,
  nss.day_of_week as new_day,
  nss.slot_number as new_slot,
  r.confirmed_at,
  a.created_at
FROM absences a
LEFT JOIN teachers t ON a.target_type = 'teacher' AND a.target_id = t.teacher_id AND t.is_current = true
LEFT JOIN students s ON a.target_type = 'student' AND a.target_id = s.student_id AND s.is_current = true
JOIN schedule_slots ss ON a.original_slot_id = ss.slot_id
LEFT JOIN reschedules r ON a.absence_id = r.absence_id
LEFT JOIN schedule_slots nss ON r.new_slot_id = nss.slot_id
WHERE a.classroom_id = :classroom_id
  AND (:target_type IS NULL OR a.target_type = :target_type)
  AND (:status IS NULL OR a.status = :status)
  AND a.created_at BETWEEN :start_date AND :end_date
ORDER BY a.created_at DESC;
```

## 5. UI状態遷移

```
[振替対応画面]
    ├─→ 欠席登録タブ
    │     ├─→ 対象者選択 → [コマ一覧表示]
    │     └─→ コマ選択 → [欠席登録フォーム]
    │                      ├─→ 登録（振替希望あり） → [振替候補提案]
    │                      │                          ├─→ 候補選択 → [確認] → [確定]
    │                      │                          └─→ 後で決める → [振替待ちとして保存]
    │                      └─→ 登録（振替希望なし） → [欠席のみ登録]
    └─→ 履歴タブ
          ├─→ 一覧表示 → [F093]
          ├─→ 行クリック（振替待ち） → [振替候補提案]
          └─→ フィルター → [一覧更新]
```

## 7. エラーコード

| コード | メッセージ | 条件 |
