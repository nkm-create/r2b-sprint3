## 3. DBマッピング

### 3.1 関連テーブル

| テーブル | 役割 |
|---------|------|
| classrooms | 教室マスタ |
| user_classrooms | ユーザー×教室関連 |
| schedules | 時間割（充足率キャッシュ） |
| terms | ターム |
| teachers | 講師マスタ（カウント用） |
| students | 生徒マスタ（カウント用） |
| notifications | 通知 |

### 3.2 エリア判定ロジック

**担当教室の取得**:
```sql
SELECT c.classroom_id, c.classroom_name, c.area_id
FROM classrooms c
WHERE c.area_id IN (
  SELECT DISTINCT area_id
  FROM user_classrooms
  WHERE user_id = :user_id
    AND deleted_at IS NULL
)
AND c.deleted_at IS NULL;
```

## 4. 機能詳細

### F020: マネジメントダッシュボード表示

#### API仕様

```
GET /api/areas/{area_id}/dashboard

クエリパラメータ:
- term_id: タームID（省略時は最新の確定ターム）

レスポンス:
{
  "summary": {
    "classroom_count": 8,
    "average_fulfillment_rate": 82.5,
    "problem_classroom_count": 2,
    "total_teachers": 200,
    "total_students": 640
  },
  "classrooms": [...],  // F021の教室一覧
  "alerts": [...]       // F023のアラート
}
```

#### DBマッピング

**サマリー集計**:
```sql
SELECT
  COUNT(DISTINCT c.classroom_id) as classroom_count,
  AVG(s.fulfillment_rate) as average_fulfillment_rate,
  COUNT(DISTINCT CASE WHEN s.fulfillment_rate < 60 THEN c.classroom_id END) as problem_classroom_count,
  SUM(t.teacher_count) as total_teachers,
  SUM(st.student_count) as total_students
FROM classrooms c
LEFT JOIN (
  SELECT s.classroom_id, s.fulfillment_rate
  FROM schedules s
  JOIN terms t ON s.term_id = t.term_id
  WHERE t.term_id = :term_id
    AND s.status = 'confirmed'
) s ON c.classroom_id = s.classroom_id
LEFT JOIN (
  SELECT classroom_id, COUNT(*) as teacher_count
  FROM teachers
  WHERE is_current = true AND deleted_at IS NULL
  GROUP BY classroom_id
) t ON c.classroom_id = t.classroom_id
LEFT JOIN (
  SELECT classroom_id, COUNT(*) as student_count
  FROM students
  WHERE is_current = true AND deleted_at IS NULL
  GROUP BY classroom_id
) st ON c.classroom_id = st.classroom_id
WHERE c.area_id = :area_id
  AND c.deleted_at IS NULL;
```

### F021: 教室別充足率一覧表示

#### 出力項目

| 項目名 | DBカラム/計算 | 説明 |
|--------|-------------|------|
| 教室名 | classrooms.classroom_name | 教室の名称 |
| 充足率 | schedules.fulfillment_rate | 1対2充足率（%） |
| ステータス | 計算 | 正常/警告/危険 |
| 前月比 | 計算 | 充足率の増減 |
| 講師数 | COUNT(teachers) | 有効な講師の人数 |
| 生徒数 | COUNT(students) | 有効な生徒の人数 |
| 時間割ステータス | schedules.status | 確定/ドラフト/未作成 |
| アラート | 計算 | 問題がある場合にバッジ表示 |

#### API仕様

```
GET /api/areas/{area_id}/classrooms

クエリパラメータ:
- term_id: タームID
- filter[status]: normal / warning / danger / all
- filter[has_alert]: true / false
- sort: fulfillment_rate / -fulfillment_rate（デフォルト）
- page, per_page: ページネーション

レスポンス:
{
  "data": [
    {
      "classroom_id": "uuid",
      "classroom_name": "渋谷教室",
      "fulfillment_rate": 85.5,
      "status": "normal",
      "previous_month_rate": 82.0,
      "change": 3.5,
      "teacher_count": 25,
      "student_count": 80,
      "schedule_status": "confirmed",
      "has_alert": false,
      "alerts": []
    }
  ],
  "pagination": {...}
}
```

#### SQL例

```sql
SELECT
  c.classroom_id,
  c.classroom_name,
  COALESCE(current_s.fulfillment_rate, 0) as fulfillment_rate,
  COALESCE(prev_s.fulfillment_rate, 0) as previous_month_rate,
  COALESCE(current_s.fulfillment_rate, 0) - COALESCE(prev_s.fulfillment_rate, 0) as change,
  COALESCE(t_count.count, 0) as teacher_count,
  COALESCE(s_count.count, 0) as student_count,
  COALESCE(current_s.status, 'none') as schedule_status
FROM classrooms c
LEFT JOIN (
  SELECT s.classroom_id, s.fulfillment_rate, s.status
  FROM schedules s
  WHERE s.term_id = :current_term_id
    AND s.status IN ('confirmed', 'draft')
  ORDER BY s.status = 'confirmed' DESC, s.version DESC
  LIMIT 1
) current_s ON c.classroom_id = current_s.classroom_id
LEFT JOIN (
  SELECT s.classroom_id, s.fulfillment_rate
  FROM schedules s
  WHERE s.term_id = :previous_term_id
    AND s.status = 'confirmed'
) prev_s ON c.classroom_id = prev_s.classroom_id
LEFT JOIN (
  SELECT classroom_id, COUNT(*) as count
  FROM teachers
  WHERE is_current = true AND deleted_at IS NULL
  GROUP BY classroom_id
) t_count ON c.classroom_id = t_count.classroom_id
LEFT JOIN (
  SELECT classroom_id, COUNT(*) as count
  FROM students
  WHERE is_current = true AND deleted_at IS NULL
  GROUP BY classroom_id
) s_count ON c.classroom_id = s_count.classroom_id
WHERE c.area_id = :area_id
  AND c.deleted_at IS NULL
ORDER BY fulfillment_rate ASC;
```

#### ステータス判定基準

| 充足率 | ステータス | 表示色 |
|--------|-----------|--------|
| 80%以上 | normal | 緑 |
| 60〜79% | warning | 黄 |
| 60%未満 | danger | 赤 |

### F022: 教室詳細ドリルダウン

#### API仕様

```
GET /api/classrooms/{classroom_id}/dashboard

レスポンス:
P002ダッシュボードと同様の構造（参照モード）
```

#### 出力セクション

| セクション | DBテーブル | 説明 |
|-----------|----------|------|
| サマリー | schedules | 充足率、講師数、生徒数 |
| 充足率推移グラフ | schedules（過去6ターム） | 過去6ターム分の推移 |
| 人員ヒートマップ | teacher_shift_preferences, student_preferences | 曜日×時限の配置状況 |
| 時間割サマリー | schedule_slots | 現在の時間割概要 |
| 問題点リスト | notifications | 充足率低下要因等 |

#### 制約

- **参照モードのみ**: 編集操作は不可
- 時間割の調整、マスタの編集等はできない

### F023: 問題教室アラート

#### アラート条件

| 条件 | notification_type | severity | 判定SQL |
|------|------------------|----------|---------|
| 充足率60%未満 | area_fulfillment_low | critical | fulfillment_rate < 60 |
| 前月比10ポイント以上低下 | area_fulfillment_drop | critical | change <= -10 |
| 充足率60〜79% | area_fulfillment_warning | warning | fulfillment_rate BETWEEN 60 AND 79 |
| 未確定の時間割が期限超過 | area_deadline_passed | warning | terms.end_date < NOW() AND status != 'confirmed' |

#### API仕様

```
GET /api/areas/{area_id}/alerts

レスポンス:
{
  "alerts": [
    {
      "classroom_id": "uuid",
      "classroom_name": "渋谷教室",
      "alert_type": "area_fulfillment_low",
      "severity": "critical",
      "message": "充足率が60%未満です（現在: 55%）",
      "created_at": "2025-03-24T10:00:00Z"
    }
  ]
}
```

## 5. UI状態遷移

```
[マネジメントダッシュボード表示]
    ├─→ 教室行クリック → [教室詳細パネル（参照モード）]
    │                        ├─→ ×ボタン → [パネル閉じる]
    │                        └─→ パネル外クリック → [パネル閉じる]
    ├─→ チェックボックスで複数選択 → [比較ボタン有効化]
    │                                  └─→ 比較ボタンクリック → [比較モーダル表示]
    │                                                            └─→ 閉じる → [一覧へ]
    ├─→ アラートクリック → [該当教室詳細パネルへ]
    ├─→ フィルター/ソート → [一覧更新]
    └─→ 期間セレクター変更 → [一覧更新]
```
