## 4. 機能詳細

### F060: 希望データマトリクス表示

#### 希望値の定義

| 値 | DBカラム値（講師） | DBカラム値（生徒） | 講師での意味 | 生徒での意味 | 表示色 |
|----|------------------|------------------|------------|------------|--------|
| ○ | available | preferred | 勤務可能 | 受講希望 | 緑 |
| × | unavailable | unavailable | 勤務不可 | 受講不可 | 赤 |
| △ | - | possible | - | できれば可 | 黄 |

#### API仕様

```
GET /api/classrooms/{classroom_id}/terms/{term_id}/preferences

クエリパラメータ:
- type: teacher / student
- page, per_page: ページネーション

レスポンス:
{
  "type": "teacher",
  "matrix": [
    {
      "person_id": "T001",
      "person_name": "田村 悠樹",
      "slots": [
        {"day_of_week": "mon", "slot_number": 1, "value": "available", "is_manually_edited": false},
        {"day_of_week": "mon", "slot_number": 2, "value": "unavailable", "is_manually_edited": true},
        ...
      ]
    }
  ],
  "time_slots": {
    "weekday": [
      {"slot_number": 1, "start_time": "16:00", "end_time": "17:20"},
      ...
    ],
    "saturday": [...]
  },
  "pagination": {...}
}
```

#### SQL例（講師）

```sql
SELECT
  t.teacher_id as person_id,
  t.teacher_name as person_name,
  tsp.day_of_week,
  tsp.slot_number,
  tsp.preference_value as value,
  tsp.is_manually_edited
FROM teachers t
LEFT JOIN teacher_shift_preferences tsp
  ON t.teacher_id = tsp.teacher_id
  AND tsp.term_id = :term_id
WHERE t.classroom_id = :classroom_id
  AND t.is_current = true
  AND t.deleted_at IS NULL
ORDER BY t.teacher_name, tsp.day_of_week, tsp.slot_number;
```

### F062: 希望データ編集

#### API仕様

```
PUT /api/classrooms/{classroom_id}/terms/{term_id}/preferences

リクエスト:
{
  "type": "teacher",
  "updates": [
    {
      "person_id": "T001",
      "day_of_week": "mon",
      "slot_number": 1,
      "value": "unavailable"
    }
  ]
}

成功レスポンス (200 OK):
{
  "message": "希望データを更新しました",
  "updated_count": 1
}
```

#### トランザクション処理

```sql
BEGIN;

-- 希望データを更新（講師の場合）
UPDATE teacher_shift_preferences SET
  preference_value = :value,
  is_manually_edited = true,
  updated_at = NOW()
WHERE term_id = :term_id
  AND teacher_id = :person_id
  AND day_of_week = :day_of_week
  AND slot_number = :slot_number;

-- 存在しない場合は挿入
INSERT INTO teacher_shift_preferences (
  preference_id, term_id, classroom_id, teacher_id,
  day_of_week, slot_number, preference_value, is_manually_edited
)
VALUES (
  gen_random_uuid(), :term_id, :classroom_id, :person_id,
  :day_of_week, :slot_number, :value, true
)
ON CONFLICT (term_id, teacher_id, day_of_week, slot_number)
DO UPDATE SET
  preference_value = :value,
  is_manually_edited = true,
  updated_at = NOW();

-- 監査ログ
INSERT INTO audit_logs (user_id, action, entity_type, entity_id, old_value, new_value)
VALUES (:user_id, 'UPDATE', 'Preference', :preference_id, :old_value, :new_value);

COMMIT;
```

#### バリデーション

| 項目 | ルール | エラーコード |
|------|--------|------------|
| 希望値 | 講師: available/unavailable、生徒: preferred/possible/unavailable | VAL_060 |
| 曜日 | 教室の営業曜日に含まれる | VAL_061 |
| 時限 | 曜日種別に応じた時限数以内 | VAL_062 |

### F063: Google Form同期

#### API仕様

```
POST /api/classrooms/{classroom_id}/terms/{term_id}/preferences/sync

リクエスト:
{
  "type": "teacher",
  "overwrite_manual_edits": false
}

成功レスポンス (200 OK):
{
  "message": "同期が完了しました",
  "summary": {
    "added": 15,
    "updated": 8,
    "skipped": 3,
    "skipped_reason": "手動編集済みのためスキップ"
  },
  "last_sync_at": "2025-03-24T10:00:00Z"
}

競合がある場合 (200 OK with conflicts):
{
  "message": "同期を完了しましたが、一部競合があります",
  "summary": {...},
  "conflicts": [
    {
      "person_id": "T001",
      "person_name": "田村 悠樹",
      "day_of_week": "mon",
      "slot_number": 1,
      "current_value": "available",
      "new_value": "unavailable",
      "is_manually_edited": true
    }
  ],
  "require_confirmation": true
}
```

#### 競合確認後の上書きAPI

```
POST /api/classrooms/{classroom_id}/terms/{term_id}/preferences/sync/confirm

リクエスト:
{
  "type": "teacher",
  "overwrite_ids": ["T001_mon_1", "T002_tue_2"]
}
```

#### トランザクション処理

```sql
BEGIN;

-- Google Formからのデータを取得（アプリケーション側で処理）

-- 既存の手動編集データを確認
SELECT person_id, day_of_week, slot_number, preference_value
FROM teacher_shift_preferences
WHERE term_id = :term_id
  AND classroom_id = :classroom_id
  AND is_manually_edited = true;

-- 手動編集がないデータのみ更新
UPDATE teacher_shift_preferences SET
  preference_value = :new_value,
  updated_at = NOW()
WHERE term_id = :term_id
  AND teacher_id = :person_id
  AND day_of_week = :day_of_week
  AND slot_number = :slot_number
  AND is_manually_edited = false;

-- 新規データの挿入
INSERT INTO teacher_shift_preferences (...)
VALUES (...)
ON CONFLICT DO NOTHING;

-- 同期日時を更新
UPDATE google_form_connections SET
  last_sync_at = NOW()
WHERE classroom_id = :classroom_id
  AND data_type = :type;

COMMIT;
```

### F064: 未回答者確認

#### API仕様

```
GET /api/classrooms/{classroom_id}/terms/{term_id}/preferences/unanswered

クエリパラメータ:
- type: teacher / student / all

レスポンス:
{
  "unanswered": [
    {
      "type": "teacher",
      "person_id": "T005",
      "person_name": "高橋 一郎",
      "registered_at": "2025-03-01",
      "status": "no_data"
    },
    {
      "type": "student",
      "person_id": "S010",
      "person_name": "山本 花子",
      "registered_at": "2025-03-15",
      "status": "partial",
      "filled_slots": 10,
      "total_slots": 24
    }
  ]
}
```

#### SQL例

```sql
-- 未回答の講師を取得
SELECT
  'teacher' as type,
  t.teacher_id as person_id,
  t.teacher_name as person_name,
  t.created_at as registered_at,
  CASE
    WHEN COUNT(tsp.preference_id) = 0 THEN 'no_data'
    WHEN COUNT(tsp.preference_id) < (
      SELECT COUNT(*) FROM time_slots ts
      JOIN classroom_settings cs ON ts.classroom_id = cs.classroom_id
      WHERE cs.classroom_id = :classroom_id
    ) THEN 'partial'
    ELSE 'complete'
  END as status,
  COUNT(tsp.preference_id) as filled_slots
FROM teachers t
LEFT JOIN teacher_shift_preferences tsp
  ON t.teacher_id = tsp.teacher_id
  AND tsp.term_id = :term_id
WHERE t.classroom_id = :classroom_id
  AND t.is_current = true
  AND t.deleted_at IS NULL
GROUP BY t.teacher_id, t.teacher_name, t.created_at
HAVING status != 'complete';
```

## 5. UI状態遷移

```
[希望データ確認画面]
    ├─→ タブ切り替え → [講師/生徒マトリクス表示]
    ├─→ セルクリック → [値トグル] → [編集状態]
    │                                ├─→ 保存 → [確定]
    │                                └─→ リセット → [編集前に戻す]
    ├─→ Google Form同期ボタン → [同期処理]
    │                            ├─→ 差分なし → [完了メッセージ]
    │                            └─→ 差分あり → [上書き確認] → [更新]
    └─→ 未回答者確認ボタン → [未回答者リスト表示]
                               └─→ 行クリック → [該当者の希望データ編集画面へ]
```

## 7. エラーコード

| コード | メッセージ | 条件 |
|--------|----------|------|
| VAL_060 | 無効な希望値です | 範囲外の値 |
| VAL_061 | 営業曜日に含まれない曜日です | 営業日外 |
| VAL_062 | 時限番号が範囲外です | 時限数超過 |
| BIZ_060 | Google Form連携が設定されていません | 未設定で同期 |
| BIZ_061 | 確定済みタームの希望データは編集できません | 確定後編集 |
