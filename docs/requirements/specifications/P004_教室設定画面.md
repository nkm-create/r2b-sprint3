# 仕様書: 教室設定画面

## 基本情報

| 項目 | 内容 |
|-----|------|
| ページID | P004 |
| ページ名 | 教室設定画面 |
| 対象ユーザー | 教室長 |

## 目的

教室の基本設定（時間枠、曜日、キャパシティなど）を管理する。初回セットアップ時および設定変更時に使用する。

## 機能一覧

| 機能ID | 機能 | 説明 | 優先度 |
|--------|------|------|--------|
| F030 | 時間枠設定 | 各限の時間帯を設定（平日1〜4限、土曜0〜4限） | Must |
| F031 | 営業曜日設定 | 教室の営業曜日を設定 | Must |
| F032 | キャパシティ設定 | 同時に実施できるコマ数（ブース数）を設定 | Must |
| F033 | デフォルト設定適用 | 標準的な設定をワンクリックで適用 | Should |
| F034 | Google Form連携設定 | 講師シフト希望・生徒受講希望データの連携設定 | Must |
| - | 科目マスタ管理 | 指導可能な科目の一覧を管理 | - |
| - | 設定保存 | 変更内容を保存 | - |

## 備考

- 初回セットアップ時はデフォルト設定を推奨
- 時間枠変更は既存の時間割に影響するため、警告を表示
- 土曜は0限（午後早い時間）から開始
- 科目マスタは全教室で共通（本部管理）の場合と、教室独自の場合がある

## 機能詳細

### F030: 時間枠設定

| 項目名 | 型 | 説明 |
|--------|-----|------|
| 限の数 | number | 1日あたりの時限数（平日: 4限、土曜: 5限） |
| 各限の時間 | object[] | 開始時刻〜終了時刻 |

### F031: 営業曜日設定

月〜土の中から営業日を選択（日曜は固定休み）

### F032: キャパシティ設定

| 項目名 | 型 | 説明 |
|--------|-----|------|
| ブース数 | number | 同時実施可能なコマ数の上限 |

### F033: デフォルト設定適用

標準的な教室設定テンプレートを適用

### F034: Google Form連携設定

#### 入力項目

| 項目名 | 型 | 必須 | バリデーション | 説明 |
|--------|-----|------|--------------|------|
| データ種別 | enum | ○ | "講師シフト希望" / "生徒受講希望" | 連携データの種別 |
| スプレッドシートURL | string | ○ | Google Sheets URL形式 | Google Formの回答が記録されるスプレッドシートのURL |
| シート名 | string | ○ | 存在するシート名 | 対象シート名 |
| カラムマッピング | object | ○ | 必須項目が全てマッピングされていること | 各列と項目の対応付け |

#### データ種別

| 種別 | 説明 | 必須マッピング項目 |
|------|------|------------------|
| 講師シフト希望 | 講師の勤務可能時間帯を収集 | 講師名、曜日×限の希望（○/×） |
| 生徒受講希望 | 生徒の受講希望時間帯を収集 | 生徒名、曜日×限の希望（○/×/△） |

#### カラムマッピング設定

| 項目 | 説明 | 例 |
|------|------|-----|
| 名前列 | 講師名または生徒名が入力されている列 | A列（氏名） |
| 希望データ列 | 曜日×限ごとの希望値が入力されている列 | B列〜Z列（月1限、月2限、...） |
| タイムスタンプ列 | 回答日時（任意） | 自動検出 |

**自動検出機能**: スプレッドシートのヘッダー行を解析し、曜日・限を含む列名から自動的にマッピングを提案

#### OAuth認証フロー

1. 「Googleアカウントで認証」ボタンをクリック
2. Googleログイン画面にリダイレクト
3. 必要な権限（スプレッドシート読み取り）を許可
4. 認証完了後、教室設定画面に戻る
5. 認証ステータスが「認証済み」に更新

#### 接続テスト

| テスト項目 | 成功条件 | エラーメッセージ |
|-----------|---------|----------------|
| 認証確認 | OAuth認証済み | Googleアカウントの認証が必要です |
| URL形式確認 | 有効なGoogle Sheets URL | 無効なスプレッドシートURLです |
| アクセス権確認 | 読み取り権限あり | スプレッドシートへのアクセス権がありません |
| シート存在確認 | 指定シートが存在 | 指定されたシートが見つかりません |
| データ取得確認 | ヘッダー行の取得成功 | データを取得できませんでした |

#### 設定フロー

```
1. データ種別を選択
   └─ 「講師シフト希望」または「生徒受講希望」を選択

2. OAuth認証
   └─ 未認証の場合、「Googleアカウントで認証」ボタンをクリック
   └─ 認証済みの場合、スキップ

3. スプレッドシートURLを入力
   └─ Google FormのレスポンスシートのURLをペースト

4. シート名を選択
   └─ URLからシート一覧を取得し、ドロップダウンで選択

5. カラムマッピング設定
   └─ 「自動検出」ボタンでヘッダー行から自動マッピング
   └─ 必要に応じて手動で調整

6. 接続テスト
   └─ 「接続テスト」ボタンで設定を検証
   └─ 成功: 緑色のチェックマークと「接続成功」メッセージ
   └─ 失敗: 赤色のエラーメッセージと原因表示

7. 設定保存
   └─ 保存ボタンで連携設定を確定
```

#### エラーハンドリング

| エラー条件 | メッセージ | 対処法 |
|-----------|----------|--------|
| 認証トークン期限切れ | 認証の有効期限が切れました。再認証してください | 再度OAuth認証を実行 |
| スプレッドシート削除 | スプレッドシートが見つかりません | URLを再確認、または新しいURLを設定 |
| 権限取り消し | アクセス権が取り消されました | Google側で権限を再付与 |
| API制限超過 | リクエスト数の上限に達しました | 時間をおいて再試行 |
