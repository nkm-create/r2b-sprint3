# 仕様書: 生徒管理画面

## 基本情報

| 項目 | 内容 |
|-----|------|
| ページID | P006 |
| ページ名 | 生徒管理画面 |
| 対象ユーザー | 教室長 |

## 目的

教室に通う生徒の情報を登録・編集・管理する。

## 機能一覧

| 機能ID | 機能 | 説明 |
|--------|------|------|
| F050 | 生徒一覧表示 | 登録済み生徒の一覧を表示 |
| F051 | 生徒新規登録 | 新しい生徒を登録 |
| F052 | 生徒情報編集 | 既存生徒の情報を編集 |
| F053 | 生徒削除 | 生徒を削除（論理削除） |
| F054 | 生徒CSVインポート | CSVファイルから生徒を一括登録 |
| F055 | 生徒検索・フィルター | 生徒名、学年、科目などで絞り込み |

## UI要素

| 要素 | 説明 | 位置 |
|------|------|------|
| ヘッダー | 共通ヘッダー | 上部固定 |
| サイドバー | 共通サイドバー | 左側固定 |
| 検索バー | 生徒名で検索 | メインエリア上部左 |
| フィルターボタン | 学年・科目・志望レベルなどでフィルター | 検索バー横 |
| 新規登録ボタン | 生徒登録モーダルを開く | メインエリア上部右 |

| 生徒一覧テーブル | 生徒情報の一覧 | メインエリア中部 |
| 編集ボタン | 各行の編集ボタン | テーブル内 |
| 削除ボタン | 各行の削除ボタン | テーブル内 |
| ページネーション | ページ切り替え | テーブル下部 |

## 生徒一覧テーブルの表示項目

| 項目 | 説明 |
|------|------|
| 生徒ID | システム内の一意ID |
| 氏名 | 生徒の氏名 |
| 学年 | 小1〜高3 |
| 受講科目 | 科目とコマ数を表示 |
| 週コマ数 | 週あたりの総コマ数 |
| 希望講師 | 希望する講師名（設定時） |
| 志望レベル | A/B/C |
| 通塾目的 | 高校受験/中学受験など |
| ステータス | 在籍/休会/退会 |

## 生徒登録・編集モーダルの項目

### 基本情報

| 項目 | 型 | 必須 | 説明 |
|------|-----|-----|------|
| 生徒ID | 自動採番 | - | システムが自動付与 |
| 氏名 | テキスト | ○ | 生徒の氏名 |
| 学年 | 選択 | ○ | 小1〜高3から選択 |

### 受講情報

| 項目 | 型 | 必須 | 説明 |
|------|-----|-----|------|
| 科目別コマ数 | 科目×コマ数リスト | ○ | 受講する科目と週あたりコマ数（複数設定可） |
| 最大連続コマ | 数値 | ○ | 連続して受講できる最大コマ数 |

#### 科目別コマ数の入力形式

科目を選択し、その科目の週あたりコマ数を指定する。複数科目を追加可能。

例：
- 中受算数: 1コマ
- 小学国語: 1コマ
- 中受理科: 1コマ

### 講師希望

| 項目 | 型 | 必須 | 説明 |
|------|-----|-----|------|
| 希望講師 | 講師選択 | - | 特定の講師を希望する場合に指定 |
| NG講師 | 講師複数選択 | - | 担当不可の講師を指定（複数可） |
| 講師希望性別 | 選択 | - | 男/女/指定なし |

### マッチング条件

| 項目 | 型 | 必須 | 説明 |
|------|-----|-----|------|
| 志望レベル | 選択 | - | A/B/C（講師の大学ランクとマッチング） |
| 通塾目的 | 選択 | - | 高校受験/中学受験/内部進学/補習/その他 |

## 機能とUIの結びつき

| 機能 | UI要素 | インタラクション |
|------|--------|----------------|
| 生徒一覧表示 | 生徒一覧テーブル | 一覧表示、ソート可能 |
| 生徒新規登録 | 新規登録ボタン | クリックでモーダル表示、入力後保存 |

| 志望レベル | A/B/C/すべて |
| 通塾目的 | 高校受験/中学受験/内部進学/補習/その他/すべて |
| 講師希望性別 | 男/女/指定なし/すべて |
| ステータス | 在籍/休会/退会/すべて |

## 備考

- 削除は論理削除（過去の時間割データとの整合性維持）
- CSVインポート時はプレビュー表示してから確定
- 科目別コマ数は「科目:コマ数」形式で保存（例: 中受算数:1,小学国語:1）
- NG講師は講師マスタから選択（相互関係：講師側のNG生徒とも連動）
- 学年と科目の整合性チェック（例: 小学生に高校科目は設定不可）
