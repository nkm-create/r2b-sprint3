# 仕様書: 振替対応画面

## 基本情報

| 項目 | 内容 |
|-----|------|
| ページID | P010 |
| ページ名 | 振替対応画面 |
| 対象ユーザー | 教室長 |

## 目的

日々発生する欠席連絡に対応し、振替候補の提案を受けて確定する。

## 機能一覧

| 機能ID | 機能 | 説明 |
|--------|------|------|
| F090 | 欠席登録 | 欠席するコマを登録 |
| F091 | 振替候補提案 | システムが振替可能な候補を提案 |
| F092 | 振替確定 | 提案された候補から選択して確定 |
| F093 | 欠席・振替履歴表示 | 過去の欠席・振替履歴を表示 |

## UI要素

| 要素 | 説明 | 位置 |
|------|------|------|
| ヘッダー | 共通ヘッダー | 上部固定 |
| サイドバー | 共通サイドバー | 左側固定 |
| ターム選択セレクター | 対象タームを選択 | メインエリア上部 |
| 欠席登録ボタン | 欠席登録モーダルを開く | メインエリア上部右 |
| カレンダービュー | 現在の時間割表示 | メインエリア左側 |
| 欠席コマハイライト | 欠席登録されたコマを強調表示 | カレンダービュー内 |
| 振替候補パネル | システムが提案する振替候補一覧 | メインエリア右側 |
| 候補選択ラジオ | 振替先候補を選択 | 振替候補パネル内 |
| 振替確定ボタン | 選択した候補で振替を確定 | 振替候補パネル下部 |
| 履歴タブ | 欠席・振替履歴への切り替え | メインエリア上部 |
| 履歴テーブル | 過去の欠席・振替一覧 | 履歴タブ選択時 |

## 機能とUIの結びつき

| 機能 | UI要素 | インタラクション |
|------|--------|----------------|
| ターム選択 | ターム選択セレクター | 選択でカレンダー表示を切り替え |
| 欠席登録 | 欠席登録ボタン、カレンダービュー | ボタンまたはカレンダー上のコマクリックで登録 |
| 振替候補提案 | 振替候補パネル | 欠席登録後に自動で候補表示 |
| 振替確定 | 候補選択ラジオ、振替確定ボタン | 候補選択後ボタンクリックで確定 |
| 欠席・振替履歴表示 | 履歴タブ、履歴テーブル | タブクリックで履歴表示 |
| カレンダービュー表示 | カレンダービュー | 時間割をグリッド形式で表示 |

## 欠席登録モーダルの項目

| 項目 | 必須 | 説明 |
|------|------|------|
| 対象者種別 | ○ | 講師 / 生徒 |
| 対象者 | ○ | 欠席する講師または生徒を選択 |
| 欠席日時 | ○ | 欠席するコマ（日付・時間枠） |
| 欠席理由 | - | 任意で理由を入力 |
| 振替希望有無 | ○ | 振替を希望するかどうか |

## 遷移

| 遷移元 | 遷移先 | トリガー |
|--------|--------|----------|
| P002 ダッシュボード | - | クイックアクション「振替対応」クリック |
| - | P002 ダッシュボード | サイドバーで他画面選択 |

## 備考

- 振替候補は講師・生徒双方の空き状況、制約条件を考慮して提案
- 振替候補がない場合はその旨を表示、手動調整を促す
- 講師欠席の場合は代替講師の提案も行う
