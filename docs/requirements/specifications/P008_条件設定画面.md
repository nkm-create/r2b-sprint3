# 仕様書: 条件設定画面

## 基本情報

| 項目 | 内容 |
|-----|------|
| ページID | P008 |
| ページ名 | 条件設定画面 |
| 対象ユーザー | 教室長 |

## 目的

時間割作成のためのターム選択と制約条件を設定する。自然言語で条件を入力し、システムが制約条件として整理・提示する。

## 機能一覧

| 機能ID | 機能 | 説明 |
|--------|------|------|
| F070 | ターム選択・作成 | 新規タームの作成または既存タームの選択 |
| F071 | 条件入力（自然言語） | 自然言語で時間割の条件を入力 |
| F072 | 条件整理・構造化 | システムが入力内容を制約条件として整理・表示 |
| F073 | 条件編集・削除 | 整理された条件を確認・修正・削除 |

## 条件の種類

実際のデータ属性に基づく条件カテゴリ:

### 講師制約

| 制約種別 | 説明 | 例 |
|----------|------|-----|
| 科目制約 | 講師の担当科目を限定 | 「佐藤先生は中受算数のみ担当」 |
| コマ数制約 | 講師の週あたりコマ数を制限 | 「田中先生は週3〜5コマ」 |
| 連続コマ制約 | 連続して担当できるコマ数を制限 | 「山田先生の連続授業は2コマまで」 |
| NG生徒制約 | 特定の生徒を担当不可に設定 | 「鈴木先生は〇〇くんNG」 |

### 生徒制約

| 制約種別 | 説明 | 例 |
|----------|------|-----|
| 講師希望 | 特定の講師を希望 | 「山田くんは佐藤先生希望」 |
| NG講師制約 | 特定の講師を担当不可に設定 | 「田中さんは〇〇先生NG」 |
| 性別希望 | 講師の性別を指定 | 「鈴木さんは女性講師希望」 |
| 連続コマ制約 | 連続して受講できるコマ数を制限 | 「伊藤くんの連続受講は2コマまで」 |

### マッチング制約

| 制約種別 | 説明 | 例 |
|----------|------|-----|
| 志望レベル | 生徒の志望レベルと講師の大学ランクをマッチング | 「志望レベルA生徒には大学ランクA/B講師を優先」 |
| 受験経験 | 中学受験・高校受験経験者を優先配置 | 「中受科目には中学受験経験講師を優先」 |
| 通塾目的 | 目的に応じた講師配置 | 「中学受験生徒には中受経験講師」 |

### 時間制約

| 制約種別 | 説明 | 例 |
|----------|------|-----|
| ブースキャパシティ | 同時実施コマ数の上限 | 「同時8コマまで」 |
| 1対2優先 | 1対2配置を優先 | 「1対2充足率90%以上を目標」 |
| 同一科目ペアリング | 同じ科目の生徒をペアに | 「可能な限り同一科目で1対2を組む」 |

## 自然言語入力例

実際の業務で使われる条件例:

- 「佐藤先生は中受算数と中受理科のみ担当」
- 「山田くんは佐藤先生を希望」
- 「田中さんはNG講師として鈴木先生を設定」
- 「中学受験生徒には中学受験経験のある講師を優先配置」
- 「志望レベルAの生徒には大学ランクA/Bの講師を配置」
- 「高校12英語は高3英語と同じ講師に担当させない」
- 「女性講師希望の生徒には女性講師を配置」
- 「1対2の充足率を90%以上にする」
- 「同一科目の生徒同士で1対2を組む」

## 備考

- 自然言語入力はAIが解析し、構造化された制約条件に変換
- システムが整理した条件は、構造化された形式（講師制約、生徒制約、マッチング制約、時間制約）で表示
- 再生成の場合、前回の条件をベースに差分のみ編集可能
- 条件の優先度を設定可能（必須条件 / 推奨条件）
- 矛盾する条件がある場合は警告を表示
